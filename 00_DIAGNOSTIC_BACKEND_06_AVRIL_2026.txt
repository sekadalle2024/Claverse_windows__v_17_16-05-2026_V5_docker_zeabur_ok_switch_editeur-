ÉTAPES DE DIAGNOSTIC:
---------------------

1️⃣  DÉMARRER LE BACKEND
   
   Commande:
   .\start-claraverse.ps1
   
   Attendre les messages:
   - "Backend demarre (Job ID: X)"
   - "Frontend demarre (Job ID: Y)"
   - "CLARAVERSE DEMARRE AVEC SUCCES!"

2️⃣  VÉRIFIER QUE LE BACKEND RÉPOND
   
   Commande:
   Invoke-WebRequest -Uri "http://localhost:5000/health" -Method GET
   
   Résultat attendu:
   StatusCode: 200
   
   Si erreur:
   - Le backend n'a pas démarré correctement
   - Voir les logs: Receive-Job -Id <BackendJobId> -Keep

3️⃣  TESTER L'IMPORT DU MODULE (avec conda)
   
   Commande:
   cd py_backend
   conda run -n claraverse_backend python test_import_etats_financiers.py
   
   OU (si pas conda):
   cd py_backend
   python test_import_etats_financiers.py
   
   Résultat attendu:
   ✅ Module etats_financiers importé avec succès
   ✅ Cache global disponible: True

4️⃣  TESTER L'IMPORT D'UNE BALANCE
   
   Action:
   1. Ouvrir http://localhost:5173
   2. Aller dans le menu "Démarrer"
   3. Cliquer sur "États Financiers"
   4. Importer: py_backend/P000 -BALANCE DEMO N_N-1_N-2.xls
   5. Observer les logs du backend
   
   Logs attendus dans le backend:
   📊 Stockage des balances dans le cache global pour l'export...
   ✅ Balances stockées dans le cache: N=XXX lignes

5️⃣  TESTER L'EXPORT DE LA LIASSE
   
   Action:
   1. Après avoir importé la balance
   2. Cliquer sur "Exporter la liasse officielle"
   3. Observer les logs du backend
   
   Logs attendus:
   📊 Récupération des balances depuis le cache global...
   ✅ Balance N récupérée: XXX lignes
   📊 Enrichissement du Bilan ACTIF avec BRUT et AMORTISSEMENT...
