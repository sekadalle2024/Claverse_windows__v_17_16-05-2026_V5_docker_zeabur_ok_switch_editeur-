🔍 POURQUOI CE PROBLÈME ?
═══════════════════════════════════════════════════════════════════

Votre fichier docker-compose.yml définit 2 services:

┌─────────────────────────────────────────────────────────────────┐
│ docker-compose.yml                                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  services:                                                      │
│    backend:          ← Service pour le backend Python          │
│      build: ./py_backend                                        │
│      ports: 8080:8080                                           │
│                                                                 │
│    frontend:         ← Service pour le frontend React          │
│      build: .                                                   │
│      ports: 80:80                                               │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

Zeabur doit savoir QUEL service builder pour chaque déploiement.

Le paramètre "Service Name" indique à Zeabur:
"Build le service X du docker-compose.yml"

Si vous mettez Service Name = "frontend" pour le Backend:
→ Zeabur va builder le service "frontend" du docker-compose.yml
→ Résultat: Backend build le frontend ! ❌

C'est comme commander une pizza et recevoir un burger ! 🍕→🍔

🆘 SI LE PROBLÈME PERSISTE
═══════════════════════════════════════════════════════════════════

1. VÉRIFIER LES LOGS COMPLETS
   → Aller dans Zeabur → Service Backend → Logs
   → Copier les 50 premières lignes
   → Chercher les erreurs

2. FORCER UN REBUILD COMPLET
   → Zeabur Dashboard → Service Backend
   → Settings → Danger Zone
   → "Delete Service" puis recréer

3. UTILISER L'ANCIEN REPO BACKEND
   → Créer un nouveau service dans Zeabur
   → Pointer vers votre ancien repo backend (py_backend seul)
   → Build Method = Dockerfile (pas Docker Compose)
   → Port = 8080

4. VÉRIFIER LES VARIABLES D'ENVIRONNEMENT
   → Backend doit avoir: CORS_ORIGINS=https://prclaravi.zeabur.app
   → Frontend doit avoir: VITE_BACKEND_URL=https://pybackend.zeabur.app
