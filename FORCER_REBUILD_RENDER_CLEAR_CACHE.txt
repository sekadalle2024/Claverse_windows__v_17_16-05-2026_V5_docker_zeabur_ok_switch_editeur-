🎯 OBJECTIF

Forcer Render à rebuild complètement en effaçant le cache.
Cela garantit que runtime.txt sera détecté.

═══════════════════════════════════════════════════════════

📋 ÉTAPES DÉTAILLÉES

1️⃣  OUVRIR RENDER DASHBOARD

   URL: https://dashboard.render.com/

2️⃣  SÉLECTIONNER VOTRE SERVICE

   - Chercher votre service backend dans la liste
   - Cliquer dessus pour ouvrir les détails

3️⃣  CLIQUER SUR "MANUAL DEPLOY"

   - Bouton en haut à droite de la page
   - Un menu déroulant s'ouvre

4️⃣  SÉLECTIONNER "CLEAR BUILD CACHE & DEPLOY"

   ⚠️  IMPORTANT: Choisir cette option, PAS "Deploy latest commit"
   
   Cette option:
   - Efface le cache de build
   - Force une installation complète
   - Garantit la détection de runtime.txt

5️⃣  CONFIRMER

   - Cliquer sur "Yes, clear cache and deploy"
   - Le rebuild commence immédiatement

6️⃣  ATTENDRE LE BUILD

   - Durée: 5-10 minutes
   - Suivre les logs en temps réel

7️⃣  VÉRIFIER LES LOGS

   Chercher cette ligne:
   "==> Using Python version 3.11.9"
   
   ✅ Si vous voyez 3.11.9 → Succès!
   ❌ Si vous voyez 3.14.3 → Problème persiste

═══════════════════════════════════════════════════════════

═══════════════════════════════════════════════════════════

⚠️  SI LE PROBLÈME PERSISTE

Après "Clear build cache & deploy", si Render utilise toujours
Python 3.14.3:

CAUSES POSSIBLES:

1. runtime.txt n'est PAS sur GitHub
   → Vérifier: https://github.com/sekadalle2024/Back-end-python-V0_03_03_2026
   → Recommencer upload si absent

2. runtime.txt est mal placé
   → Doit être à la RACINE du repository
   → PAS dans un sous-dossier

3. runtime.txt a un nom incorrect
   → Doit être exactement "runtime.txt"
   → Tout en minuscules

4. runtime.txt a un contenu incorrect
   → Doit contenir: python-3.11.9
   → Pas d'espace, pas de ligne vide
