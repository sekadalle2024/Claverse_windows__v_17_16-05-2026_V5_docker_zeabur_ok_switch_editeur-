PROBLÈMES IDENTIFIÉS
═══════════════════════════════════════════════════════════════════════════

1. CYCLE HARDCODÉ
───────────────────────────────────────────────────────────────────────────
Le cycle était hardcodé sur "Trésorerie" dans le prompt de l'agent RAG.

AVANT:
- Quel est le cycle concerné ? → Trésorerie

PROBLÈME:
L'agent RAG cherchait toujours des comptes de trésorerie, même si
l'utilisateur spécifiait un autre cycle (RH, Ventes, Achats, etc.)

2. USER MESSAGE NON VISIBLE
───────────────────────────────────────────────────────────────────────────
Les variables utilisées ne pointaient pas vers les bonnes données.

AVANT:
*** [user_message] = "{{JSON.stringify($json.data, null, 2)}}"
** [messagz] = "{{JSON.stringify($json.Code integrate7, null, 2)}}"

PROBLÈME:
- $json.data pointe vers les données brutes du webhook
- $json.Code integrate7 n'existe pas
- L'agent RAG ne voyait pas le message utilisateur structuré

SOLUTIONS APPLIQUÉES
═══════════════════════════════════════════════════════════════════════════

SOLUTION 1: INDEXER LE CYCLE SUR [Cycle]
───────────────────────────────────────────────────────────────────────────

AVANT (hardcodé):
```
- Quel est le cycle concerné ? → Trésorerie
```

APRÈS (dynamique):
```
- Quel est le cycle concerné ? → {{ $json.data[0].User_message.find(item => item['[Cycle]'])?.['[Cycle]'] || 'Non spécifié' }}
```

EXPLICATION:
- $json.data[0].User_message : Array des clés-valeurs du message utilisateur
- .find(item => item['[Cycle]']) : Cherche l'objet contenant [Cycle]
- ?.['[Cycle]'] : Extrait la valeur de [Cycle]
- || 'Non spécifié' : Fallback si [Cycle] n'est pas trouvé

SOLUTION 2: CORRIGER LES VARIABLES USER MESSAGE
───────────────────────────────────────────────────────────────────────────

AVANT (incorrect):
```
[user_message] = "{{JSON.stringify($json.data, null, 2)}}"
[messagz] = "{{JSON.stringify($json.Code integrate7, null, 2)}}"
```

APRÈS (correct):
```
[user_message] = {{ JSON.stringify($json.data[0].User_message, null, 2) }}

[cycle_detecte] = {{ $json.data[0].User_message.find(item => item['[Cycle]'])?.['[Cycle]'] || 'Non spécifié' }}

[constat] = {{ $json.data[0].User_message.find(item => item['[Constat]'])?.['[Constat]'] || 'Non spécifié' }}

[anomalie] = {{ $json.data[0].User_message.find(item => item['[Anomalie]'])?.['[Anomalie]'] || 'Non spécifié' }}
```

EXPLICATION:
- $json.data[0].User_message : Pointe vers le message structuré par Code integrate7
- Variables séparées pour cycle, constat, anomalie
- Facilite l'utilisation dans les queries RAG

SOLUTION 3: ADAPTER LES QUERIES RAG AU CYCLE
───────────────────────────────────────────────────────────────────────────

AVANT (hardcodé sur trésorerie):
```
RECHERCHE 1:
Query → "compte caisse trésorerie SYSCOHADA fonctionnement débit crédit"
```

APRÈS (dynamique selon le cycle):
```
RECHERCHE 1 - COMPTE PRINCIPAL DU CYCLE :
Query → "compte {{ $json.data[0].User_message.find(item => item['[Cycle]'])?.['[Cycle]'] || 'trésorerie' }} SYSCOHADA fonctionnement débit crédit"
```

EXPLICATION:
- La query s'adapte automatiquement au cycle spécifié
- Si [Cycle] = "RH" → cherche "compte RH SYSCOHADA..."
- Si [Cycle] = "Ventes" → cherche "compte Ventes SYSCOHADA..."
- Fallback sur "trésorerie" si [Cycle] non spécifié
