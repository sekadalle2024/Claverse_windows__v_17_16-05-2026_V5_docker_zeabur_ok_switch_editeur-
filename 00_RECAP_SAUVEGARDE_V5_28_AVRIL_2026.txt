PROBLÈME 1: "Execution de scripts est désactivé"
───────────────────────────────────────────────────────────────────────────
ERREUR:
    cannot be loaded because running scripts is disabled

SOLUTION:
    Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass

Puis relancer le script.


PROBLÈME 2: HTTP 408 Timeout
───────────────────────────────────────────────────────────────────────────
ERREUR:
    error: RPC failed; HTTP 408 curl 22

SOLUTION:
1. Utiliser le script de commits multiples:
   .\push-commits-multiples-v5.ps1

2. Ou utiliser GitHub Desktop


PROBLÈME 3: Connection Reset
───────────────────────────────────────────────────────────────────────────
ERREUR:
    error: RPC failed; curl 55 Send failure: Connection was reset

SOLUTION:
1. Vérifier la connexion Internet
2. Attendre 5 minutes
3. Relancer le script (il reprendra où il s'est arrêté)


PROBLÈME 4: Repository distant incorrect
───────────────────────────────────────────────────────────────────────────
ERREUR:
    fatal: repository not found

SOLUTION:
Le script configure automatiquement le bon repository.
Si l'erreur persiste:

    git remote set-url origin https://github.com/sekadalle2024/Claverse_windows__v_14_28-04-2026_V5_docker_zeabur_ok_switch_editeur-.git


PROBLÈME 5: Branche incorrecte
───────────────────────────────────────────────────────────────────────────
SOLUTION:
Vérifier la branche actuelle:
    git branch

Changer de branche si nécessaire:
    git checkout main
    # ou
    git checkout master
