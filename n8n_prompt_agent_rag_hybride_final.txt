[INSTRUCTIONS DE RECHERCHE RAG]

ÉTAPE 1 — ANALYSE SÉMANTIQUE DES DONNÉES

Utilise les 4 variables essentielles pour comprendre le contexte :

1. CYCLE → Quel domaine comptable ?
   - Trésorerie → Comptes 5xx (banque, caisse)
   - Ventes → Comptes 7xx (produits) + 4xx (clients)
   - Achats → Comptes 6xx (charges) + 4xx (fournisseurs)
   - RH → Comptes 66x (charges personnel) + 42x (personnel)
   - Immobilisations → Comptes 2xx (immobilisations)

2. REFERENCE → Quelle opération exacte ?
   - Analyser la description pour identifier les flux économiques
   - Identifier les comptes principaux mouvementés

3. ASSERTION → Quel type de contrôle ?
   - Existence → Vérifier que l'actif/passif existe réellement
   - Exhaustivité → Vérifier que toutes les opérations sont enregistrées
   - Validité → Vérifier les pièces justificatives
   - Exactitude → Vérifier les montants et calculs
   - Imputation → Vérifier l'affectation aux bons comptes

4. ANOMALIE → Quelle régularisation ?
   - Analyser l'anomalie pour déterminer l'écriture de correction
   - Identifier les comptes à débiter/créditer pour corriger

ÉTAPE 2 — RECHERCHES RAG OBLIGATOIRES

Lance les recherches suivantes dans la vector store :

RECHERCHE 1 - COMPTE PRINCIPAL DU CYCLE :
Query → "compte {{ $json.cycle || "comptable" }} SYSCOHADA fonctionnement débit crédit {{ $json.reference || "" }}"
Objectif → Identifier le compte principal du cycle, son sens, ses contreparties

RECHERCHE 2 - COMPTES DE CHARGES/PRODUITS :
Query → "compte charges produits {{ $json.cycle || "comptable" }} SYSCOHADA {{ $json.reference || "" }}"
Objectif → Identifier le(s) compte(s) de charges ou produits mouvementés

RECHERCHE 3 - TVA SI APPLICABLE :
Query → "TVA récupérable déductible {{ $json.cycle || "comptable" }} SYSCOHADA compte 445"
Objectif → Identifier le compte de TVA applicable si pertinent

RECHERCHE 4 - ÉCRITURE DE RÉGULARISATION SELON ASSERTION :
Query → "écriture régularisation extourne {{ $json.cycle || "comptable" }} SYSCOHADA ajustement audit {{ $json.assertion || "" }} {{ $json.anomalie || "" }}"
Objectif → Identifier le schéma d'extourne comptable pour corriger l'anomalie

RECHERCHE 5 (si nécessaire) - COMPTES TIERS :
Query → "compte tiers fournisseur client {{ $json.cycle || "comptable" }} SYSCOHADA classe 4"
Objectif → Vérifier si un compte tiers (classe 4) doit apparaître

RECHERCHE 6 (si assertion = validité/existence) - PIÈCES JUSTIFICATIVES :
Query → "pièces justificatives {{ $json.cycle || "comptable" }} SYSCOHADA contrôle {{ $json.assertion || "" }}"
Objectif → Identifier les pièces justificatives requises selon l'assertion testée
