ANALYSE DU FORMAT ACTUEL (py_backend/etats_controle_exhaustifs.py)
═══════════════════════════════════════════════════════════════════════════════

STRUCTURE RETOURNÉE:
```python
{
    'titre': 'Etat de contrôle Bilan Actif',
    'postes': [
        {'ref': 'CA', 'libelle': 'Total Actif', 'montant_n': 1000000, 'montant_n1': 950000},
        {'ref': 'CB', 'libelle': 'Nombre de postes (N)', 'montant_n': 50, 'montant_n1': 0},
        {'ref': 'CC', 'libelle': 'Nombre de postes (N-1)', 'montant_n': 0, 'montant_n1': 48},
        {'ref': 'CD', 'libelle': 'Variation Total', 'montant_n': 50000, 'montant_n1': 0},
    ]
}
```

PROBLÈMES:
1. ❌ Certaines lignes ont montant_n1 = 0 (ex: ligne CB)
2. ❌ Certaines lignes ont montant_n = 0 (ex: ligne CC)
3. ❌ Les variations sont sur une seule colonne (montant_n)
4. ❌ Format pas optimal pour l'affichage en 2 colonnes

FORMAT ATTENDU (test_etats_controle_html.html)

STRUCTURE ATTENDUE:
```python
{
    'titre': 'Etat de contrôle Bilan Actif',
    'postes': [
        {'ref': 'CA', 'libelle': 'Total Actif', 'montant_n': 1000000, 'montant_n1': 950000},
        {'ref': 'CB', 'libelle': 'Nombre de postes', 'montant_n': 50, 'montant_n1': 48},
        {'ref': 'CD', 'libelle': 'Variation Total', 'montant_n': 50000, 'montant_n1': 0},
    ]
}
```

AVANTAGES:
1. ✅ Toutes les lignes ont les deux colonnes remplies (sauf variations)
2. ✅ Plus compact et lisible
3. ✅ Cohérent avec le format liasse (2 colonnes N et N-1)
4. ✅ Facilite la comparaison visuelle
