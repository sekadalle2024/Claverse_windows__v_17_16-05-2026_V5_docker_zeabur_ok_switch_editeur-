================================================================================
DIAGNOSTIC - ERREUR "FAILED TO FETCH" PERSISTANTE
Date: 06 Avril 2026
================================================================================

SITUATION:
----------
L'erreur "failed to fetch" persiste malgré la correction appliquée (cache global).

CAUSES POSSIBLES:
-----------------

1️⃣  BACKEND NON REDÉMARRÉ
    Le backend DOIT être complètement arrêté et redémarré pour prendre en compte
    les modifications du code Python.
    
    Symptôme: Les modifications ne sont pas prises en compte
    Solution: Arrêter complètement et redémarrer le backend

2️⃣  ERREUR DE SYNTAXE PYTHON
    Une erreur de syntaxe dans etats_financiers.py empêche le module de charger.
    
    Symptôme: Le backend démarre mais l'endpoint ne fonctionne pas
    Solution: Tester l'import du module avec test_import_etats_financiers.py

3️⃣  ERREUR DANS UN AUTRE MODULE
    L'erreur peut venir d'un module importé (tableau_flux_tresorerie, annexes_liasse, etc.)
    
    Symptôme: Erreur lors de l'import ou de l'exécution
    Solution: Vérifier les logs du backend

4️⃣  PROBLÈME DE CORS OU RÉSEAU
    Le frontend ne peut pas communiquer avec le backend.
    
    Symptôme: Erreur "failed to fetch" même si le backend fonctionne
    Solution: Vérifier que le backend écoute sur http://localhost:8000

5️⃣  ERREUR DANS LE TRAITEMENT DES DONNÉES
    L'erreur se produit pendant le traitement de la balance, pas au démarrage.
    
    Symptôme: Le backend démarre OK mais plante lors du traitement
    Solution: Vérifier les logs du backend lors de l'import de la balance

================================================================================
SOLUTIONS PAR CAUSE
================================================================================

SOLUTION 1: Backend non redémarré
----------------------------------
1. Arrêter le backend (Ctrl+C)
2. Attendre que le processus se termine
3. Redémarrer: .\start-claraverse.ps1
4. Attendre "✅ Backend démarré sur http://localhost:8000"
5. Réessayer l'import de la balance

SOLUTION 2: Erreur de syntaxe
------------------------------
1. Exécuter: python py_backend/test_import_etats_financiers.py
2. Lire le message d'erreur
3. Corriger l'erreur dans etats_financiers.py
4. Redémarrer le backend
5. Réessayer

SOLUTION 3: Erreur dans un autre module
----------------------------------------
1. Vérifier les logs du backend au démarrage
2. Identifier le module qui cause l'erreur
3. Corriger l'erreur dans ce module
4. Redémarrer le backend
5. Réessayer

SOLUTION 4: Problème CORS/Réseau
---------------------------------
1. Vérifier que le backend écoute sur localhost:8000
2. Vérifier que le frontend pointe vers localhost:8000
3. Vérifier les paramètres CORS dans main.py
4. Redémarrer le backend
5. Rafraîchir le frontend (Ctrl+F5)

SOLUTION 5: Erreur pendant le traitement
-----------------------------------------
1. Observer les logs du backend lors de l'import
2. Identifier la ligne qui cause l'erreur
3. Corriger l'erreur
4. Redémarrer le backend
5. Réessayer
