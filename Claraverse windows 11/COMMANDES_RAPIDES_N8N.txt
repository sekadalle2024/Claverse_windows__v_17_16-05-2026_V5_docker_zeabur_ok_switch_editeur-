🔍 TEST DANS LA CONSOLE DU NAVIGATEUR
--------------------------------------
Copiez-collez ce code dans la console (F12):

fetch('https://xomava4845.app.n8n.cloud/webhook/htlm_processor', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ question: '<table><tr><td>Test</td></tr></table>' })
})
.then(r => {
  console.log('📊 Statut:', r.status, r.statusText);
  return r.text();
})
.then(text => {
  console.log('📏 Taille:', text.length, 'caractères');
  console.log('📄 Contenu:', text);
  if (text.length === 0) {
    console.error('❌ PROBLÈME: Réponse vide !');
    console.log('👉 Solution: Ajoutez un node "Respond to Webhook" dans n8n');
  } else {
    try {
      const json = JSON.parse(text);
      console.log('✅ JSON valide:', json);
    } catch (e) {
      console.error('❌ PROBLÈME: Pas du JSON !');
      console.log('👉 Solution: Configurez le node pour retourner du JSON');
    }
  }
})
.catch(e => {
  console.error('❌ ERREUR:', e.message);
  console.log('👉 Vérifiez l\'URL et les permissions CORS');
});

🎯 INDICATEURS CLÉS À VÉRIFIER
-------------------------------

1. TAILLE REÇUE:
   ✅ > 0 caractères = Données reçues
   ❌ 0 caractères = Réponse vide (problème n8n)

2. PARSING JSON:
   ✅ SUCCÈS = Format correct
   ❌ ÉCHEC = Pas du JSON (problème format)

3. STATUT HTTP:
   ✅ 200 OK = Connexion réussie
   ❌ 404/500 = Problème serveur


💡 ASTUCE
---------
Le nombre de caractères est l'indicateur le plus important !
Si vous voyez "0 caractères", c'est que n8n ne renvoie rien.


🆘 PROBLÈMES FRÉQUENTS
----------------------

PROBLÈME: "Taille reçue: 0 caractères"
CAUSE: Pas de node "Respond to Webhook" dans n8n
SOLUTION: Ajoutez ce node à la fin de votre workflow

PROBLÈME: "Parsing: ❌ ÉCHEC"
CAUSE: n8n retourne du texte brut, pas du JSON
SOLUTION: Dans "Respond to Webhook", sélectionnez "JSON"

PROBLÈME: "Erreur CORS"
CAUSE: n8n bloque les requêtes depuis votre domaine
SOLUTION: Configurez CORS dans les settings n8n
