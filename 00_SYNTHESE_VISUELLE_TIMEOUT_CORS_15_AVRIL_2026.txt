📋 ANALYSE COMPLÈTE EFFECTUÉE:

┌─────────────────────────────────────────────────────────────────────────────┐
│ POURQUOI L'ERREUR APRÈS 3 MINUTES MALGRÉ UN TIMEOUT DE 10 MINUTES ?        │
└─────────────────────────────────────────────────────────────────────────────┘

1️⃣ TIMEOUTS MULTIPLES DANS LE CODE
   ─────────────────────────────────────────────────────────────────────────
   ⚠️ Le timeout de 10 minutes n'est PAS le seul timeout
   
   Timeouts de 30 secondes trouvés dans:
   • NodeRegistry.ts (ligne 504)
   • FlowExecutor.ts (ligne 30)
   • constants.ts (ligne 247)
   • nodeDefinitions.ts (ligne 380)
   • simpleNodeDefinitions.ts (ligne 1133)
   
   Impact: Ces timeouts peuvent court-circuiter le timeout principal

2️⃣ TIMEOUTS DU NAVIGATEUR
   ─────────────────────────────────────────────────────────────────────────
   Les navigateurs ont leurs propres limites:
   
   • Chrome/Edge: ~5 minutes
   • Firefox: ~90 secondes (1.5 minutes)
   • Safari: ~60 secondes (1 minute)
   
   Impact: Le navigateur peut annuler la requête avant 10 minutes

3️⃣ TIMEOUTS DU SERVEUR N8N
   ─────────────────────────────────────────────────────────────────────────
   n8n a ses propres timeouts configurables:
   
   • Timeout de workflow
   • Timeout de webhook
   • Timeout de node HTTP
   
   Impact: Si n8n timeout avant 10 minutes, la requête échoue

4️⃣ TIMEOUTS RÉSEAU/PROXY
   ─────────────────────────────────────────────────────────────────────────
   Les proxies et load balancers ont leurs propres timeouts:
   
   • Nginx: 60 secondes par défaut
   • Apache: 300 secondes (5 minutes)
   • Cloudflare: 100 secondes
   
   Impact: Un timeout intermédiaire peut couper la connexion

5️⃣ PROBLÈME CORS
   ─────────────────────────────────────────────────────────────────────────
   Les requêtes CORS font d'abord une requête OPTIONS (preflight)
   
   Si le preflight timeout ou échoue:
   • La requête principale n'est jamais envoyée
   • Erreur "Failed to fetch" ou "Network error"
   
   Impact: Configuration CORS incorrecte peut bloquer les requêtes

┌─────────────────────────────────────────────────────────────────────────────┐
│ CONCLUSION                                                                  │
└─────────────────────────────────────────────────────────────────────────────┘

Le timeout de 10 minutes dans claraApiService.ts est correct, MAIS il peut être
court-circuité par:

1. Un timeout plus court dans le code (30 secondes)
2. Le timeout du navigateur (60-300 secondes)
3. Le timeout du serveur n8n (configurable)
4. Un timeout d'un proxy/load balancer (60-300 secondes)
5. Un problème de configuration CORS

Le timeout le plus court prend toujours le dessus.
