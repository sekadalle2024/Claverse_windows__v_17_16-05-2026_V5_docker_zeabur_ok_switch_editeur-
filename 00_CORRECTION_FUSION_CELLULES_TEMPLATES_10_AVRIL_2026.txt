  Date: 10 Avril 2026

📋 PROBLÈME IDENTIFIÉ
─────────────────────────────────────────────────────────────────────────────

Les cellules des colonnes étaient fusionnées dans les templates de tables pour
certains cases:

1. Cases 40-43: Cellules "Intitule" fusionnées
2. Cases 36-37: Cellules "Colonne A" fusionnées  
3. Case 38: Cellules "Domaines" fusionnées

🔍 CAUSE RACINE
─────────────────────────────────────────────────────────────────────────────

Le problème n'était PAS dans les données JSON, mais dans la conversion Markdown.
Les cellules vides (même avec un espace " ") étaient traitées comme vides par
le parseur Markdown et fusionnées visuellement par le rendu HTML/CSS.

✅ SOLUTION APPLIQUÉE (VERSION 2)
─────────────────────────────────────────────────────────────────────────────

1. Modification de la fonction convertArrayTableToMarkdown():
   - Les cellules vides sont maintenant converties en tirets "-"
   - Cela force le rendu Markdown à afficher des cellules distinctes

2. Retour aux chaînes vides "" dans les templates JSON:
   - Plus besoin d'espaces " " qui ne fonctionnaient pas
   - La fonction de conversion gère automatiquement les cellules vides

📝 FICHIER MODIFIÉ
─────────────────────────────────────────────────────────────────────────────

src/services/claraApiService.ts

🔧 MODIFICATIONS TECHNIQUES
─────────────────────────────────────────────────────────────────────────────

1. Fonction convertArrayTableToMarkdown (ligne ~767):
   ```typescript
   // CORRECTION FUSION CELLULES: Utiliser un tiret pour les cellules vides
   // au lieu d'un espace qui peut être ignoré par le rendu Markdown
   return cleanValue || "-";
   ```

2. Templates JSON (cases 36-43):
   - Toutes les valeurs vides sont maintenant "" (chaînes vides)
   - La fonction convertArrayTableToMarkdown les convertit en "-"

─────────────────────────────────────────────────────────────────────────────

rendu HTML/CSS à traiter chaque cellule comme ayant du contenu, empêchant
ainsi la fusion visuelle des cellules.
