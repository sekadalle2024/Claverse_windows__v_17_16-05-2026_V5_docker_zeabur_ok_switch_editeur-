🎯 MISSION
═══════════════════════════════════════════════════════════════════════════════

Résoudre le problème où l'action "Exporter template Excel" n'exportait que
la table active au lieu de toutes les tables présentes dans la div.

📊 ANALYSE DU PROBLÈME
═══════════════════════════════════════════════════════════════════════════════

CONTEXTE:
─────────────────────────────────────────────────────────────────────────────
Dans le menu contextuel (menu.js), l'action "Exporter template Excel" est
destinée à exporter toutes les tables contenues dans la div de la table active.

PROBLÈME CONSTATÉ:
─────────────────────────────────────────────────────────────────────────────
❌ Seule la table active est exportée en Excel
❌ Les autres tables dans la div ne sont pas exportées

HYPOTHÈSES VALIDÉES:
─────────────────────────────────────────────────────────────────────────────
✅ Hypothèse 1: Chaque table dans la div est elle-même contenue dans une div
✅ Hypothèse 2: La fonctionnalité n'était pas implémentée pour exporter toutes
   les tables, mais uniquement la table active

SOLUTION IDENTIFIÉE:
─────────────────────────────────────────────────────────────────────────────
✅ Utiliser le code de l'action "Export Synthèse CAC" qui fonctionne déjà
   correctement pour détecter toutes les tables dans la div
