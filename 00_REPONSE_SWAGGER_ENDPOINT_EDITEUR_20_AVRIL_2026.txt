❓ Question: L'endpoint n'apparaît pas dans Swagger, est-ce normal?
✅ Réponse: OUI, c'est normal dans certains cas

═══════════════════════════════════════════════════════════════════════════════
  🔍 POURQUOI L'ENDPOINT N'APPARAÎT PAS DANS SWAGGER
═══════════════════════════════════════════════════════════════════════════════

RAISON 1: TAG NON CONFIGURÉ DANS FASTAPI
─────────────────────────────────────────────────────────────────────────────

Le router utilise le tag "editeur":
  router = APIRouter(prefix="/editeur", tags=["editeur"])

Mais FastAPI peut ne pas afficher ce tag dans Swagger si:
  • Le tag n'est pas dans la liste des tags principaux
  • Le tag n'a pas de métadonnées (description, etc.)

RAISON 2: ORDRE D'AFFICHAGE
─────────────────────────────────────────────────────────────────────────────

Swagger affiche les endpoints par ordre alphabétique des tags.
Le tag "editeur" peut être en bas de la liste ou masqué.

RAISON 3: CONFIGURATION SWAGGER
─────────────────────────────────────────────────────────────────────────────

FastAPI peut avoir une configuration Swagger qui filtre certains tags.

  ✅ COMMENT VÉRIFIER QUE L'ENDPOINT FONCTIONNE
═══════════════════════════════════════════════════════════════════════════════

MÉTHODE 1: TESTER DIRECTEMENT L'ENDPOINT
─────────────────────────────────────────────────────────────────────────────

Exécuter le script de test:
  .\test-endpoint-editeur-zeabur.ps1

Ou tester manuellement:

  # Health Check
  curl https://pybackend.zeabur.app/editeur/health

  # Endpoint Process
  curl -X POST https://pybackend.zeabur.app/editeur/process \
    -H "Content-Type: application/json" \
    -d '{"command": "editeur", "message": "test"}'

MÉTHODE 2: VÉRIFIER OPENAPI.JSON
─────────────────────────────────────────────────────────────────────────────

Aller sur:
  https://pybackend.zeabur.app/openapi.json

Chercher "/editeur" dans le JSON.
Si présent → L'endpoint est bien enregistré.

MÉTHODE 3: VÉRIFIER LES LOGS ZEABUR
─────────────────────────────────────────────────────────────────────────────

Chercher dans les logs:
  "✅ Endpoint Editeur router loaded successfully"

Si présent → Le router est bien chargé.

═══════════════════════════════════════════════════════════════════════════════
  🔧 COMMENT FAIRE APPARAÎTRE L'ENDPOINT DANS SWAGGER
═══════════════════════════════════════════════════════════════════════════════

OPTION 1: AJOUTER DES MÉTADONNÉES AU TAG
─────────────────────────────────────────────────────────────────────────────

Dans main.py, ajouter:

  tags_metadata = [
      {
          "name": "editeur",
          "description": "Endpoint de test pour le switch backend",
      },
      # ... autres tags
  ]

  app = FastAPI(
      title="Clara Backend API",
      version="1.0.0",
      openapi_tags=tags_metadata
  )

OPTION 2: UTILISER UN TAG EXISTANT
─────────────────────────────────────────────────────────────────────────────

Dans endpoint_editeur.py, changer:

  router = APIRouter(prefix="/editeur", tags=["default"])

Ou utiliser un tag déjà configuré dans main.py.

OPTION 3: NE RIEN FAIRE
─────────────────────────────────────────────────────────────────────────────

Si l'endpoint fonctionne (tests réussis), ce n'est pas grave qu'il
n'apparaisse pas dans Swagger. C'est un endpoint de test interne.

ÉTAPE 1: EXÉCUTER LE SCRIPT DE TEST
─────────────────────────────────────────────────────────────────────────────

  .\test-endpoint-editeur-zeabur.ps1

Ce script va tester:
  • Health Check (/editeur/health)
  • Endpoint Process (/editeur/process)
  • Présence dans Swagger
  • Présence dans OpenAPI JSON

ÉTAPE 2: ANALYSER LES RÉSULTATS
─────────────────────────────────────────────────────────────────────────────

Si les tests 1 et 2 sont ✅ SUCCÈS:
  → L'endpoint fonctionne correctement
  → Le problème 404 est résolu
  → Peu importe s'il n'apparaît pas dans Swagger

Si les tests 1 et 2 sont ❌ ÉCHEC:
  → Le fichier n'a pas été déployé correctement
  → Vérifier les logs Zeabur
  → Forcer un rebuild manuel

  ❌ Les tests échouent
  ❌ Le frontend ne peut pas appeler l'endpoint
