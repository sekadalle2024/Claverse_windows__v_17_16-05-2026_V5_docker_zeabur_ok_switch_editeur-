SOLUTION 1: Démarrer le backend en mode interactif pour voir les logs
───────────────────────────────────────────────────────────────────────────────
1. Arrêter le backend actuel:

2. Ouvrir un nouveau terminal PowerShell

   cd py_backend
   conda activate claraverse_backend
   python main.py

4. Dans un autre terminal, tester:
   .\test-backend-simple.ps1

5. Observer les logs dans le premier terminal pour voir l'erreur exacte


SOLUTION 2: Vérifier les fichiers de log du backend
───────────────────────────────────────────────────────────────────────────────
Si le backend écrit dans un fichier de log:
   - Chercher les fichiers .log dans py_backend/
   - Lire les dernières lignes pour voir l'erreur


SOLUTION 3: Ajouter un timeout au frontend (solution temporaire)
───────────────────────────────────────────────────────────────────────────────
Même si le backend a une erreur, ajouter un timeout au fetch() permettra
d'avoir un message d'erreur plus clair dans le navigateur:

   .\fix-fetch-timeout.ps1

Puis rafraîchir le navigateur (Ctrl+F5) et retester.
Le message d'erreur du backend sera affiché dans le navigateur.
