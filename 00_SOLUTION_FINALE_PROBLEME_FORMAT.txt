🎯 PROBLÈME IDENTIFIÉ
─────────────────────────────────────────────────────────────────────────────

Le front-end affiche du texte brut au lieu de tableaux Markdown formatés :

  ❌ ACTUEL:
  **Tableau entete**:**Etape de mission**: Programme de travail...

  ✅ ATTENDU:
  | Rubrique | Description |
  |----------|-------------|
  | **Etape de mission** | Programme de travail |


🔍 CAUSE RACINE
─────────────────────────────────────────────────────────────────────────────

Le node Code dans n8n retourne :
  [{ json: { data: {...} } }]

Mais claraApiService.ts attend :
  [{ data: {...} }]

La propriété "json" supplémentaire empêche la détection du FORMAT 4 !

Dans claraApiService.ts (ligne 1245), la condition est :
  if (firstItem && typeof firstItem === "object" && "data" in firstItem)

Cette condition échoue car firstItem = { json: { data: {...} } }
donc "data" n'est PAS directement dans firstItem, mais dans firstItem.json


🔧 SOLUTION
─────────────────────────────────────────────────────────────────────────────

MODIFIER LA DERNIÈRE LIGNE DU NODE CODE dans n8n :

  ❌ ANCIEN CODE (ne fonctionne pas):
  return [{ json: { data: finalData } }];

  ✅ NOUVEAU CODE (fonctionne):
  return [{ data: finalData }];

📊 DIFFÉRENCE ENTRE LES VERSIONS

  Résultat : claraApiService.ts ne détecte pas le FORMAT 4
  car "data" n'est pas directement dans l'objet

  car "data" est directement dans l'objet


🎓 EXPLICATION TECHNIQUE
─────────────────────────────────────────────────────────────────────────────

Dans claraApiService.ts, la fonction normalizeN8nResponse() vérifie :

  const firstItem = result[0];
  if (firstItem && typeof firstItem === "object" && "data" in firstItem) {
    // FORMAT 4 détecté
    const dataContent = firstItem.data;
    contentToDisplay = this.convertStructuredDataToMarkdown(dataContent);
  }

Pour que cette condition soit vraie, il faut que :
  result[0] = { data: {...} }

Et PAS :
  result[0] = { json: { data: {...} } }

Car dans le second cas, "data" n'est pas directement dans result[0],
mais dans result[0].json


💡 POURQUOI LA PROPRIÉTÉ "json" ÉTAIT AJOUTÉE ?
─────────────────────────────────────────────────────────────────────────────

Dans n8n, la convention habituelle est de retourner :
  return [{ json: { ... } }];

Mais pour le FORMAT 4 de claraApiService.ts, il faut retourner :
  return [{ data: { ... } }];

C'est une exception à la convention n8n, mais c'est nécessaire pour que
la détection du FORMAT 4 fonctionne correctement.
