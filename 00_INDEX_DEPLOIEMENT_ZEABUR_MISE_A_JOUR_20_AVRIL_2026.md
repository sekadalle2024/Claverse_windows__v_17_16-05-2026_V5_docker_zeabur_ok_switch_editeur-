## 📞 Commandes Rapides

### Vérification Locale
```powershell
# Vérifier que nginx.conf n'existe pas
Test-Path "nginx.conf"  # Doit retourner: False

# Vérifier docker-compose.yml
Get-Content docker-compose.yml

# Vérifier les Dockerfiles
Test-Path "Dockerfile.frontend"
Test-Path "py_backend/Dockerfile"
```

### Tests Backend
```powershell
# Test health endpoint
curl https://pybackend.zeabur.app/health

# Test avec PowerShell
Invoke-RestMethod -Uri "https://pybackend.zeabur.app/health"
```

### Tests Frontend
```powershell
# Ouvrir dans le navigateur
start https://prclaravi.zeabur.app

# Test avec curl
curl -I https://prclaravi.zeabur.app
```

### CLI Zeabur (optionnel)
```powershell
# Installation
npm install -g @zeabur/cli

# Login
zeabur auth login

# Déployer
zeabur deploy --project-id=xxx --service=backend
zeabur deploy --project-id=xxx --service=frontend

# Logs
zeabur logs --project-id=xxx --service=backend --follow
```

## 🆘 Support et Aide

### Si le problème persiste

1. **Vérifier les logs complets**
   ```
   Zeabur Dashboard → Service Backend → Logs
   Copier les 50 premières lignes
   ```

2. **Forcer un rebuild complet**
   ```
   Settings → Danger Zone → Delete Service
   Puis recréer le service
   ```

3. **Utiliser l'ancien repo backend**
   ```
   Créer nouveau service
   Pointer vers ancien repo py_backend
   Build Method = Dockerfile (pas Docker Compose)
   ```

4. **Consulter la documentation Zeabur**
   - [Zeabur Docs](https://zeabur.com/docs)
   - [Docker Compose Support](https://zeabur.com/docs/deploy/docker-compose)
