💡 LEÇONS APPRISES
═══════════════════════════════════════════════════════════════════

1. DOCKER COMPOSE DANS ZEABUR:
   • Nécessite configuration précise du Service Name
   • Plus complexe avec 2 services
   • Mieux de déployer séparément

2. REPOS SÉPARÉS:
   • Plus simple pour le déploiement
   • Déploiement indépendant
   • Moins de confusion

3. BUILD METHOD:
   • "Dockerfile" = Utilise un Dockerfile spécifique
   • "Docker Compose" = Utilise docker-compose.yml
   • Zeabur exécute SOIT l'un SOIT l'autre

4. ROOT DIRECTORY:
   • Indique où chercher les fichiers
   • "/" = racine du repo
   • Important pour que le Dockerfile trouve les fichiers
