📝 CE QUE FAIT LE SCRIPT
═════════════════════════

Le script divise automatiquement votre gros commit en 6 parties:

Partie 1: Code Source React/TypeScript (src/)
Partie 2: Backend Python (py_backend/)
Partie 3: Fichiers Publics (public/)
Partie 4: Documentation principale (Doc menu demarrer/, etc.)
Partie 5: Autres documentations (*.md, *.txt, Doc_Github_Issue/, etc.)
Partie 6: Fichiers Restants (configuration, etc.)

Chaque partie fait < 30 MB → Pas de timeout !

🔧 DÉTAILS TECHNIQUES
═════════════════════

Le script effectue automatiquement:

1. Annule le gros commit (git reset --soft HEAD~1)
2. Configure Git pour gros fichiers:
   - core.compression 0
   - http.postBuffer 1048576000
   - http.lowSpeedTime 999999
   - http.lowSpeedLimit 0
3. Crée 6 commits séparés
4. Push chaque commit avec retry (3 tentatives max)
5. Vérifie le succès de chaque push
