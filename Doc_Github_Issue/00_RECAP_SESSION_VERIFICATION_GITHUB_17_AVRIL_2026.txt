🎓 EXPLICATION DU PROBLEME

Quand l'utilisateur tape:
```
curl -s "https://api.github.com/repos/..." | ConvertFrom-Json
```

PowerShell interprète:
1. `curl` = alias de `Invoke-WebRequest`
2. `-s` = paramètre inconnu (ignoré)
3. L'URL est passée comme argument positionnel
4. Mais `Invoke-WebRequest` attend `-Uri` explicitement
5. PowerShell demande: "Uri:" (paramètre manquant)

SOLUTION:
Utiliser le script qui gère tout automatiquement:
```
& ".\Doc render deploy\verifier-fichiers-github.ps1"
```
