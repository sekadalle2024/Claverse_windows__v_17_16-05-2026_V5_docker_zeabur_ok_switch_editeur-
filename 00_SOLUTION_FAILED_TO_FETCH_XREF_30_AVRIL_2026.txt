✅ SOLUTION COMPLÈTE - ERREUR "FAILED TO FETCH" XREF
=====================================================
Date: 30 avril 2026

🎯 SOLUTION IDENTIFIÉE
----------------------

Le problème "failed to fetch" vient de 2 causes principales:

1. ❌ Fichiers Google Drive NON PARTAGÉS
   → Le frontend ne peut pas accéder aux fichiers privés

2. ❌ Credentials Google Drive mal configurés ou expirés
   → Le workflow n8n ne peut pas accéder à Google Drive

📋 SOLUTION ÉTAPE PAR ÉTAPE
----------------------------

ÉTAPE 1: Mettre à jour le workflow n8n
---------------------------------------

✓ Ajouter un node "Partager Fichier" après "Rechercher Fichier"

Le nouveau workflow inclut:
- Node "Partager Fichier" (Google Drive - Share)
- Type: anyone
- Role: reader
- Cela rend le fichier accessible publiquement

✓ Fichier créé: workflow_n8n_ouvrir_xref_AVEC_PARTAGE.json

ÉTAPE 2: Importer le workflow corrigé dans n8n
-----------------------------------------------

1. Aller sur https://n8nsqlite.zeabur.app
2. Cliquer sur "Import from File"
3. Sélectionner: workflow_n8n_ouvrir_xref_AVEC_PARTAGE.json
4. Configurer les credentials Google Drive OAuth2
5. Activer le workflow

ÉTAPE 3: Configurer les credentials Google Drive
-------------------------------------------------

Dans n8n > Credentials > Google Drive OAuth2 API:

✓ Client ID: [VOTRE_GOOGLE_CLIENT_ID]
✓ Client Secret: [VOTRE_GOOGLE_CLIENT_SECRET]
✓ Scope: https://www.googleapis.com/auth/drive
✓ Auth URI Query Parameters: access_type=offline

Cliquer sur "Connect my account" et autoriser l'accès

ÉTAPE 4: Tester le workflow
----------------------------

Exécuter le script de test:

```powershell
.\test-workflow-afficher-xref.ps1
```

Résultat attendu:
- success: true
- file.shared: true
- file.webViewLink: https://drive.google.com/file/d/[ID]/view?usp=sharing
