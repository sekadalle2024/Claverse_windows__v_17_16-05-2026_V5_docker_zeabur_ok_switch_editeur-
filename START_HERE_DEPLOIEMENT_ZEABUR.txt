PROBLÈME ACTUEL:
❌ Le service "Backend" dans Zeabur build le FRONTEND au lieu du BACKEND

PREUVE:
Les logs montrent: dist/assets/*.js (fichiers JavaScript du frontend)
Au lieu de: pip install, uvicorn running (backend Python)

CAUSE:
Le paramètre "Service Name" est configuré sur "frontend" au lieu de "backend"

═══════════════════════════════════════════════════════════════════
✅ SOLUTION EN 3 ÉTAPES (5 MINUTES)
═══════════════════════════════════════════════════════════════════

┌─────────────────────────────────────────────────────────────────┐
│ ÉTAPE 1: CORRIGER LA CONFIGURATION BACKEND                     │
└─────────────────────────────────────────────────────────────────┘

1. Aller sur: https://zeabur.com/dashboard

2. Sélectionner votre projet

3. Cliquer sur le service "Backend" (pybackend.zeabur.app)

4. Aller dans "Settings" ou "Configuration"

5. Dans "Build Configuration", MODIFIER:
   
   Service Name: [frontend ▼]  ← Changer en "backend"
                 [backend  ▼]  ← ✅ CORRECT

6. Cliquer "Save" ou "Update"

7. Cliquer "Redeploy" ou "Rebuild"

┌─────────────────────────────────────────────────────────────────┐
│ ÉTAPE 2: VÉRIFIER LA CONFIGURATION FRONTEND                    │
└─────────────────────────────────────────────────────────────────┘

1. Cliquer sur le service "Frontend" (prclaravi.zeabur.app)

2. Aller dans "Settings"

3. VÉRIFIER que:
   
   Service Name: [frontend ▼]  ← Doit être "frontend"
   Port:         [80       ]   ← Doit être 80

4. Si incorrect, corriger et "Redeploy"

┌─────────────────────────────────────────────────────────────────┐
│ ÉTAPE 3: ATTENDRE ET TESTER                                    │
└─────────────────────────────────────────────────────────────────┘

1. Attendre 3-5 minutes (build en cours)

2. Vérifier les logs du BACKEND:
   ✅ Doit montrer: "pip install", "uvicorn running"
   ❌ Ne doit PAS montrer: "dist/assets/*.js"

3. Tester les URLs:
   
   Backend:  https://pybackend.zeabur.app/health
   → Doit retourner: {"status": "healthy"}
   
   Frontend: https://prclaravi.zeabur.app
   → Doit afficher l'application

═══════════════════════════════════════════════════════════════════
💡 POURQUOI CE PROBLÈME ?
═══════════════════════════════════════════════════════════════════

Le fichier docker-compose.yml définit 2 services:
• "backend"  → Build le backend Python
• "frontend" → Build le frontend React

Zeabur doit savoir QUEL service builder.

Si le service Zeabur "Backend" a Service Name = "frontend",
il va builder le mauvais service !

C'est comme dire à un chef:
"Prépare le plat 'dessert' pour l'entrée" → Erreur !

═══════════════════════════════════════════════════════════════════
🆘 BESOIN D'AIDE ?
═══════════════════════════════════════════════════════════════════

Si le problème persiste:

1. Vérifier que nginx.conf n'existe PAS à la racine du projet
   (doit être supprimé)

2. Consulter les logs complets dans Zeabur

3. Essayer de déployer les services séparément:
   - Backend: Root Directory = "py_backend"
   - Frontend: Root Directory = "/"

4. Vérifier les variables d'environnement
