❌ Erreur Build Frontend Zeabur:
   npm ci --only=production échoue
   Message: "npm-shrinkwrap.json with lockfileVersion >= 1"
   
🔍 Cause Racine:
   - npm ci nécessite un package-lock.json valide
   - Le fichier existe mais peut être incompatible avec la version npm dans le container
   - npm ci est strict et ne tolère aucune divergence

✅ SOLUTION APPLIQUÉE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📝 Modification: Dockerfile.frontend

AVANT:
   RUN npm ci --only=production

APRÈS:
   RUN npm install --production=false

💡 Pourquoi cette solution:
   ✓ npm install est plus tolérant que npm ci
   ✓ Fonctionne avec ou sans package-lock.json
   ✓ --production=false installe toutes les dépendances (dev + prod)
   ✓ Nécessaire pour le build (vite, typescript, etc.)

💡 NOTES IMPORTANTES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

⚠️  npm ci vs npm install:
    - npm ci: Rapide, strict, nécessite package-lock.json exact
    - npm install: Plus lent, tolérant, crée/met à jour package-lock.json
    
✅  Pour Docker:
    - npm install est recommandé pour éviter les problèmes de compatibilité
    - Le cache Docker compense la différence de vitesse
    
🔄  Workflow de déploiement:
    1. Développement local → npm install
    2. Push GitHub → Zeabur détecte les changements
    3. Build Docker → npm install dans le container
    4. Déploiement automatique
