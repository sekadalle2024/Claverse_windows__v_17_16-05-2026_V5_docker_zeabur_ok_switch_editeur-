CORRECTION: CONVERSION INTÉGRALE DES MATRICES DE RISQUES
Date: 02 Avril 2026
Problème: Conversion Alpha-3 → Num-5 incomplète
================================================================================

PROBLÈME IDENTIFIÉ:
-------------------
Les méthodes de conversion de matrices (convertToMatrixNum3/4/5 et 
convertToMatrixAlpha3/4/5) utilisent des méthodes normalize*() qui ne 
convertissent PAS les valeurs alphabétiques vers numériques ou vice-versa.

Exemple:
- "Faible" → Num-5 ne fonctionne pas
- "1" → Alpha-3 ne fonctionne pas

SOLUTION:
---------
1. Créer 6 nouvelles méthodes de conversion INTÉGRALE:
   - convertToNumeric3(value)  // Alpha → Num-3
   - convertToNumeric4(value)  // Alpha → Num-4
   - convertToNumeric5(value)  // Alpha → Num-5
   - convertToAlpha3(value)    // Num → Alpha-3
   - convertToAlpha4(value)    // Num → Alpha-4
   - convertToAlpha5(value)    // Num → Alpha-5

2. Modifier les 6 méthodes de conversion de matrices pour utiliser
   ces nouvelles méthodes au lieu de normalize*()

EMPLACEMENT DANS LE CODE:
--------------------------
Fichier: public/menu.js
Classe: ContextualMenuManager

Insérer les nouvelles méthodes après normalizeToNum5() (ligne ~1958)
et avant le commentaire "COULEURS POUR LES MATRICES"
