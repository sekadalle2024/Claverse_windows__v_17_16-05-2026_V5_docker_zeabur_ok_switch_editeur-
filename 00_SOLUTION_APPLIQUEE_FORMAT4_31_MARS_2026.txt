SOLUTION APPLIQUÉE - FORMAT 4 OBJET vs ARRAY
31 Mars 2026
================================================================

PROBLÈME IDENTIFIÉ:
------------------
Le webhook n8n retourne un OBJET { data: {...} } au lieu d'un ARRAY [{ data: {...} }]
→ La condition Array.isArray(result) dans claraApiService.ts échouait
→ Les tableaux Markdown n'étaient pas affichés (texte brut à la place)

CAUSE RACINE:
-------------
n8n transforme automatiquement la réponse du node Code:
- Code retourne: [{ json: { data: {...} } }]
- Webhook retourne: { data: {...} } (objet, pas array)

SOLUTION APPLIQUÉE:
-------------------
✅ Modification de src/services/claraApiService.ts (lignes ~1230-1280)

Ajout de DEUX détections FORMAT 4:

1. FORMAT 4A (NOUVEAU): Objet direct { data: {...} }
   - Détecte: result && typeof result === "object" && !Array.isArray(result) && "data" in result
   - C'est ce que le webhook n8n retourne actuellement
   - Format: programme_travail_data_object

2. FORMAT 4B (ORIGINAL): Array [{ data: {...} }]
   - Détecte: Array.isArray(result) && result[0] && "data" in result[0]
   - Au cas où n8n retournerait un array
   - Format: programme_travail_data_array

FICHIERS MODIFIÉS:
------------------
✅ src/services/claraApiService.ts
   - Ajout détection FORMAT 4A (objet direct)
   - Conservation FORMAT 4B (array)
   - Les deux formats appellent convertStructuredDataToMarkdown()
