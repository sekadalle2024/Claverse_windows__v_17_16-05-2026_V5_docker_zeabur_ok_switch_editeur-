📋 OBJECTIF
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Valider que l'export template Word exporte TOUTES les tables d'un message
et que le timeout de la bibliothèque docx est suffisant.


🎯 SCÉNARIO DE TEST
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

ÉTAPE 1: PRÉPARER UN MESSAGE AVEC PLUSIEURS TABLES
────────────────────────────────────────────────────────────────────────────

1. Ouvrir E-audit dans le navigateur
2. Créer un nouveau message avec 3 tables (ou plus)
3. Exemple de structure:

   Table 1: Risques identifiés
   ┌─────────────┬──────────┬────────────┐
   │ Risque      │ Impact   │ Probabilité│
   ├─────────────┼──────────┼────────────┤
   │ Risque A    │ Élevé    │ Moyenne    │
   │ Risque B    │ Moyen    │ Faible     │
   └─────────────┴──────────┴────────────┘

   Table 2: Actions correctives
   ┌─────────────┬──────────┬────────────┐
   │ Action      │ Délai    │ Responsable│
   ├─────────────┼──────────┼────────────┤
   │ Action 1    │ 30 jours │ Manager A  │
   │ Action 2    │ 60 jours │ Manager B  │
   └─────────────┴──────────┴────────────┘

   Table 3: Budget
   ┌─────────────┬──────────┐
   │ Poste       │ Montant  │
   ├─────────────┼──────────┤
   │ Personnel   │ 50 000 € │
   │ Matériel    │ 20 000 € │
   └─────────────┴──────────┘


ÉTAPE 2: OUVRIR LA CONSOLE DU NAVIGATEUR
────────────────────────────────────────────────────────────────────────────

1. Appuyer sur F12 (ou Ctrl+Shift+I)
2. Aller dans l'onglet "Console"
3. Effacer les logs existants (icône poubelle)


ÉTAPE 3: LANCER L'EXPORT WORD
────────────────────────────────────────────────────────────────────────────

1. Clic droit sur N'IMPORTE QUELLE table du message
2. Sélectionner "Export template Word"
3. Observer les logs dans la console

❌ ERREURS À ÉVITER
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

SI VOUS VOYEZ:
────────────────────────────────────────────────────────────────────────────

❌ "1 table(s) trouvée(s) dans le message"
   → PROBLÈME: La correction n'a pas été appliquée correctement
   → SOLUTION: Vérifier que le fichier menu.js a bien été modifié

❌ "Timeout: docx non accessible"
   → PROBLÈME: Le timeout est encore trop court
   → SOLUTION: Vérifier que ensureDocxLibraryLoaded() a été corrigé

❌ "Conteneur parent non trouvé"
   → PROBLÈME: Les tables ne sont pas dans des wrappers data-container-id
   → SOLUTION: Vérifier que wrap-tables-auto.js fonctionne

❌ "Aucune table trouvée"
   → PROBLÈME: Aucune table sélectionnée
   → SOLUTION: Cliquer d'abord sur une table avant d'ouvrir le menu


🔍 DIAGNOSTIC AVANCÉ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

VÉRIFIER LA STRUCTURE DOM:
────────────────────────────────────────────────────────────────────────────

1. Ouvrir les DevTools (F12)
2. Onglet "Elements"
3. Chercher une table dans le DOM
4. Vérifier la structure:

   ✅ STRUCTURE CORRECTE:
   <div data-container-id="container-xxx-1">
     <table>...</table>
   </div>
   <div data-container-id="container-xxx-2">
     <table>...</table>
   </div>

   ❌ STRUCTURE INCORRECTE:
   <table>...</table>
   <table>...</table>
   (pas de wrappers data-container-id)


VÉRIFIER LE CHARGEMENT DE DOCX:
────────────────────────────────────────────────────────────────────────────

Dans la console, taper:
  window.docx

✅ SI CHARGÉ:
  → Affiche: Object { Document: ..., Packer: ..., ... }

❌ SI NON CHARGÉ:
  → Affiche: undefined
  → Attendre 30 secondes et réessayer
