✅ CONFIGURATION FRONTEND (2 MINUTES)
═══════════════════════════════════════════════════════════════════

1. Zeabur Dashboard → Add Service → Git

2. Sélectionner: claraverse (repo complet)

3. Settings → Remplir:

   ┌─────────────────────────────────────────────────────────────┐
   │ Build Method:   Dockerfile                                  │
   │ Dockerfile:     Dockerfile.frontend                         │
   │ Root Directory: /                                           │
   │ Port:           80                                          │
   └─────────────────────────────────────────────────────────────┘

4. Save & Redeploy

5. Attendre 5-10 minutes

6. Tester: https://prclaravi.zeabur.app

═══════════════════════════════════════════════════════════════════
💡 RÉPONSES À VOS QUESTIONS
═══════════════════════════════════════════════════════════════════

Q1: Backend avec repo séparé ?
✅ OUI - Vous avez bien fait ! Backend fonctionne.

Q2: Frontend avec repo séparé ou complet ?
✅ Utiliser repo COMPLET (claraverse)
   → Spécifier Dockerfile.frontend
   → Root Directory = /

Q3: Docker Compose ET Dockerfile exécutés ?
❌ NON - Zeabur exécute SOIT l'un SOIT l'autre
   → Build Method = "Dockerfile" → Utilise UNIQUEMENT Dockerfile.frontend
   → docker-compose.yml sera IGNORÉ

═══════════════════════════════════════════════════════════════════
🎯 POURQUOI CETTE CONFIGURATION ?
═══════════════════════════════════════════════════════════════════

✅ Repo complet (claraverse):
   → Vous l'avez déjà sur GitHub
   → Pas besoin de créer un nouveau repo

✅ Build Method = "Dockerfile":
   → Zeabur utilise UNIQUEMENT Dockerfile.frontend
   → docker-compose.yml est ignoré
   → Comme pour le backend (même approche)

✅ Root Directory = "/":
   → Dockerfile.frontend est à la racine
   → Il a besoin d'accéder à src/, public/, package.json
