Problème: nginx: [emerg] "events" directive is not allowed here

═══════════════════════════════════════════════════════════════════
📋 DIAGNOSTIC
═══════════════════════════════════════════════════════════════════

ERREUR ZEABUR:
```
2026/04/20 11:10:43 [emerg] 1#1: "events" directive is not allowed 
here in /etc/nginx/conf.d/default.conf:1
nginx: [emerg] "events" directive is not allowed here in 
/etc/nginx/conf.d/default.conf:1
```

CAUSE IDENTIFIÉE:
✗ Fichier nginx.conf à la racine du projet
✗ Contient une configuration complète avec directive "events"
✗ Ce fichier était copié dans /etc/nginx/conf.d/default.conf
✗ La directive "events" n'est autorisée que dans nginx.conf principal

═══════════════════════════════════════════════════════════════════
✅ SOLUTION APPLIQUÉE
═══════════════════════════════════════════════════════════════════

1. SUPPRESSION DU FICHIER CONFLICTUEL
   ✓ Supprimé: nginx.conf (racine du projet)
   
2. CONFIGURATION CORRECTE DÉJÀ EN PLACE
   ✓ Dockerfile.frontend génère la bonne config
   ✓ Configuration dans /etc/nginx/conf.d/default.conf
   ✓ Sans directive "events" (correcte pour un fichier conf.d)

═══════════════════════════════════════════════════════════════════
📁 STRUCTURE NGINX CORRECTE
═══════════════════════════════════════════════════════════════════

DANS LE CONTAINER:
/etc/nginx/
├── nginx.conf                    ← Config principale (fournie par nginx:alpine)
│   └── Contient: events, http, include conf.d/*.conf
└── conf.d/
    └── default.conf              ← Notre config (générée par Dockerfile)
        └── Contient: server { ... }

RÈGLE IMPORTANTE:
• nginx.conf principal    → Peut contenir "events", "http"
• conf.d/*.conf          → Seulement "server" blocks
• Notre Dockerfile génère correctement default.conf
