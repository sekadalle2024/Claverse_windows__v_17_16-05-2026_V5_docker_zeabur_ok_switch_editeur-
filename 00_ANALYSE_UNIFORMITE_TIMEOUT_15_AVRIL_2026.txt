🎯 QUESTION
═══════════════════════════════════════════════════════════════════════════════

Le timeout est-il uniforme dans tout le code de claraApiService.ts ?

═══════════════════════════════════════════════════════════════════════════════
✅ RÉPONSE : OUI, LE TIMEOUT EST UNIFORME
═══════════════════════════════════════════════════════════════════════════════

Le timeout de 5 minutes est utilisé de manière uniforme dans TOUTES les requêtes
vers n8n, SAUF pour la fonction de test de connexion qui utilise 30 secondes.

═══════════════════════════════════════════════════════════════════════════════
📊 DÉTAIL DES TIMEOUTS
═══════════════════════════════════════════════════════════════════════════════

┌─────────────────────────────────────────────────────────────────────────────┐
│ 1. TIMEOUT PRINCIPAL (5 MINUTES)                                            │
└─────────────────────────────────────────────────────────────────────────────┘

📍 LOCALISATION:
   Ligne 330: private n8nTimeout = 5 * 60 * 1000;

⏱️ VALEUR:
   5 minutes = 300 000 millisecondes

🎯 UTILISATION:
   ✅ Ligne 1969: setTimeout(() => { controller.abort(); }, this.n8nTimeout);
   
   Cette ligne est dans la fonction sendMessage() qui est utilisée pour
   TOUTES les requêtes normales vers n8n.

📝 FONCTIONS UTILISANT CE TIMEOUT:
   • sendMessage() - Ligne 1969
   • Toutes les requêtes vers les endpoints n8n
   • Tous les workflows (Case 1 à Case 43)

┌─────────────────────────────────────────────────────────────────────────────┐
│ 2. TIMEOUT DE TEST (30 SECONDES)                                            │
└─────────────────────────────────────────────────────────────────────────────┘

📍 LOCALISATION:
   Ligne 372: const timeoutId = setTimeout(() => controller.abort(), 30000);

⏱️ VALEUR:
   30 secondes = 30 000 millisecondes

🎯 UTILISATION:
   ⚠️ Uniquement dans testN8nConnection() - Ligne 372
   
   Cette fonction est utilisée UNIQUEMENT pour tester la connexion à n8n,
   PAS pour les requêtes normales.

📝 FONCTIONS UTILISANT CE TIMEOUT:
   • testN8nConnection() - Ligne 372 (fonction de test uniquement)

LIGNE 330 - DÉCLARATION DU TIMEOUT PRINCIPAL
─────────────────────────────────────────────────────────────────────────────
private n8nTimeout = 5 * 60 * 1000; // 5 minutes par défaut

✅ Ce timeout est utilisé partout dans le code via this.n8nTimeout


LIGNE 341 - LOG DU TIMEOUT AU DÉMARRAGE
─────────────────────────────────────────────────────────────────────────────
console.log("⏱️ Timeout configuré:", this.n8nTimeout / 1000, "secondes");

✅ Affiche: "Timeout configuré: 300 secondes"


LIGNE 347-349 - MÉTHODE POUR MODIFIER LE TIMEOUT
─────────────────────────────────────────────────────────────────────────────
public setN8nTimeout(timeoutMs: number): void {
  this.n8nTimeout = timeoutMs;
  console.log("⏱️ Nouveau timeout n8n:", timeoutMs / 1000, "secondes");
}

✅ Permet de modifier le timeout dynamiquement


LIGNE 355-357 - MÉTHODE POUR RÉCUPÉRER LE TIMEOUT
─────────────────────────────────────────────────────────────────────────────
public getN8nTimeout(): number {
  return this.n8nTimeout;
}

✅ Permet de lire le timeout actuel


LIGNE 372 - TIMEOUT DE TEST (30 SECONDES)
─────────────────────────────────────────────────────────────────────────────
const timeoutId = setTimeout(() => controller.abort(), 30000);

⚠️ ATTENTION: Ce timeout de 30 secondes est DIFFÉRENT du timeout principal
⚠️ Il est utilisé UNIQUEMENT dans testN8nConnection()
⚠️ Il N'AFFECTE PAS les requêtes normales vers n8n


LIGNE 1933 - LOG DU TIMEOUT AVANT REQUÊTE
─────────────────────────────────────────────────────────────────────────────
console.log("⏱️ Timeout configuré:", this.n8nTimeout / 1000, "secondes");

✅ Affiche le timeout avant chaque requête


LIGNE 1964-1969 - UTILISATION DU TIMEOUT PRINCIPAL
─────────────────────────────────────────────────────────────────────────────
const controller = new AbortController();
const timeoutId = setTimeout(() => {
  console.warn(
    `⏱️ Timeout atteint (${this.n8nTimeout / 1000}s), annulation...`,
  );
  controller.abort();
}, this.n8nTimeout);

✅ Utilise this.n8nTimeout (5 minutes) pour TOUTES les requêtes normales


LIGNE 2078-2083 - CALCUL DU TIMEOUT POUR LE MESSAGE D'ERREUR
─────────────────────────────────────────────────────────────────────────────
const timeoutMinutes = Math.floor(this.n8nTimeout / 60000);
const timeoutSeconds = Math.floor((this.n8nTimeout % 60000) / 1000);
const timeoutDisplay =
  timeoutMinutes > 0
    ? `${timeoutMinutes} minute${timeoutMinutes > 1 ? "s" : ""}...`
    : `${timeoutSeconds} secondes`;

✅ Calcule "5 minutes" pour l'affichage dans le message d'erreur


LIGNE 2086 - MESSAGE D'ERREUR AVEC TIMEOUT

errorMessage = `## Notification système\n\n⏱️ **Délai d'attente dépassé** (>5 minutes)\n\n`;

✅ CONCLUSION

OUI, LE TIMEOUT EST UNIFORME DANS TOUT LE CODE DE CLARAAPI SERVICE

Le timeout de 5 minutes (this.n8nTimeout) est utilisé de manière cohérente
dans TOUTES les requêtes vers n8n.

Le seul timeout différent (30 secondes) est utilisé UNIQUEMENT dans la fonction
de test de connexion (testN8nConnection) et n'affecte PAS les requêtes normales.

┌─────────────────────────────────────────────────────────────────────────────┐
│ TOUTES LES REQUÊTES VERS N8N UTILISENT LE MÊME TIMEOUT: 5 MINUTES          │
└─────────────────────────────────────────────────────────────────────────────┘

═══════════════════════════════════════════════════════════════════════════════
🎯 RECOMMANDATION
═══════════════════════════════════════════════════════════════════════════════

Le timeout est uniforme et bien implémenté. Aucune modification nécessaire.

Si vous souhaitez également modifier le timeout de test (30 secondes), vous
pouvez le faire en modifiant la ligne 372:

AVANT:
const timeoutId = setTimeout(() => controller.abort(), 30000);

APRÈS (exemple avec 1 minute):
const timeoutId = setTimeout(() => controller.abort(), 60000);

Mais ce n'est PAS nécessaire car ce timeout n'affecte que les tests de connexion.
