   - Menu contextuel → Papier de travail → Ajouter Cross référence horizontale
   - OU raccourci: Ctrl+Shift+H

🔍 POINTS DE VÉRIFICATION
═══════════════════════════════════════════════════════════════════════════════

✅ 1. POSITIONNEMENT
   - Schéma de calcul est-il AU-DESSUS de la table principale ?
   - Cross référence est-elle EN DESSOUS de la table principale ?

✅ 2. NOMBRE DE COLONNES
   - Les 3 tables ont-elles le même nombre de colonnes (6) ?
   - Compter visuellement les bordures verticales

✅ 3. ALIGNEMENT HORIZONTAL
   - La dernière variable (C) est-elle alignée avec la colonne "Ecart" ?
   - La dernière référence [  ] est-elle alignée avec la colonne "Ecart" ?

✅ 4. COLONNES VIDES
   - Y a-t-il 3 colonnes vides AVANT les variables ?
   - Y a-t-il 0 colonne vide APRÈS les variables ?
   - Même chose pour les références ?

🐛 PROBLÈMES POSSIBLES
═══════════════════════════════════════════════════════════════════════════════

❌ Cross référence AU-DESSUS au lieu d'EN DESSOUS
   → Vérifier public/papier-travail-cross-ref-horizontale.js ligne 374
   → Doit utiliser insertBefore(nextSibling) ou appendChild()

❌ Nombre de colonnes différent
   → Vérifier que findEcartColumnIndex() trouve bien la colonne "Ecart"
   → Vérifier les logs dans la console (F12)

❌ Variables pas alignées sur "Ecart"
   → Vérifier calculateEmptyColumnsCount()
   → Formule: ecartIndex - nbVariables + 1

❌ Modules standalone ne fonctionnent pas
   → Vérifier que les méthodes d'alignement sont présentes
   → Vérifier que buildSchemaTable/buildCrossRefTable reçoivent tablePrincipale
