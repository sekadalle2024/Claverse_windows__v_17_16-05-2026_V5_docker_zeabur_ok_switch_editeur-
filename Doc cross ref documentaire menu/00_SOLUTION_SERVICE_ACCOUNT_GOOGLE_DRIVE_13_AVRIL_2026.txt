❌ PROBLÈME : OAuth2 ne fonctionne pas dans les nodes Code Python
═══════════════════════════════════════════════════════════════════════════════

L'erreur persiste car :
- Les credentials OAuth2 ne sont pas accessibles dans les nodes Code Python
- La fonction _get_credentials() ne fonctionne pas comme prévu
- Le token n'est pas correctement transmis

✅ SOLUTION : UTILISER UN SERVICE ACCOUNT GOOGLE
═══════════════════════════════════════════════════════════════════════════════

Un Service Account est un compte Google spécial pour les applications.
Il utilise une clé JSON au lieu d'OAuth2.

═══════════════════════════════════════════════════════════════════════════════

📋 ÉTAPE 1 : CRÉER UN SERVICE ACCOUNT
═══════════════════════════════════════════════════════════════════════════════

1. Aller sur : https://console.cloud.google.com/

2. Sélectionner votre projet (ou en créer un)

3. Menu : IAM & Admin > Service Accounts

4. Cliquer sur "Create Service Account"

5. Remplir :
   - Name: n8n-drive-uploader
   - Description: Service account pour n8n upload Google Drive
   
6. Cliquer sur "Create and Continue"

7. Rôle : Sélectionner "Editor" (ou "Drive File Admin")

8. Cliquer sur "Continue" puis "Done"

═══════════════════════════════════════════════════════════════════════════════

📋 ÉTAPE 2 : CRÉER UNE CLÉ JSON
═══════════════════════════════════════════════════════════════════════════════

1. Dans la liste des Service Accounts, cliquer sur celui créé

2. Onglet "Keys"

3. Cliquer sur "Add Key" > "Create new key"

4. Choisir "JSON"

5. Cliquer sur "Create"

6. Un fichier JSON est téléchargé automatiquement

7. IMPORTANT : Garder ce fichier en sécurité !

═══════════════════════════════════════════════════════════════════════════════

📋 ÉTAPE 3 : ACTIVER L'API GOOGLE DRIVE
═══════════════════════════════════════════════════════════════════════════════

1. Dans Google Cloud Console

2. Menu : APIs & Services > Library

3. Rechercher "Google Drive API"

4. Cliquer sur "Google Drive API"

5. Cliquer sur "Enable"

═══════════════════════════════════════════════════════════════════════════════

📋 ÉTAPE 4 : PARTAGER LE DOSSIER GOOGLE DRIVE
═══════════════════════════════════════════════════════════════════════════════

1. Ouvrir le fichier JSON téléchargé

2. Copier l'email du service account (format: xxx@xxx.iam.gserviceaccount.com)

3. Dans Google Drive, créer ou ouvrir le dossier "Dossier CAC"

4. Clic droit > Partager

5. Ajouter l'email du service account

6. Donner les droits "Éditeur"

7. Cliquer sur "Partager"

═══════════════════════════════════════════════════════════════════════════════

📋 ÉTAPE 5 : CONFIGURER N8N AVEC LE SERVICE ACCOUNT
═══════════════════════════════════════════════════════════════════════════════

OPTION A : UTILISER LE NODE "GOOGLE DRIVE" NATIF (RECOMMANDÉ)
─────────────────────────────────────────────────────────────────────────────

Au lieu des nodes Code Python, utiliser les nodes Google Drive natifs de n8n :

1. Dans n8n : Settings > Credentials > New Credential

2. Choisir "Google Service Account"

3. Coller le contenu du fichier JSON dans le champ

4. Sauvegarder avec le nom : "Google Drive Service Account"

5. Utiliser les nodes "Google Drive" au lieu de "Code Python"

═══════════════════════════════════════════════════════════════════════════════

OPTION B : UTILISER LE JSON DANS LES NODES CODE PYTHON
─────────────────────────────────────────────────────────────────────────────

Si vous voulez absolument utiliser du code Python :

1. Créer une variable d'environnement dans n8n avec le JSON

2. Dans le code Python, utiliser google-auth au lieu de requests

MAIS ATTENTION : Cette option nécessite d'installer google-auth-oauthlib
ce qui n'est pas possible dans n8n natif.
