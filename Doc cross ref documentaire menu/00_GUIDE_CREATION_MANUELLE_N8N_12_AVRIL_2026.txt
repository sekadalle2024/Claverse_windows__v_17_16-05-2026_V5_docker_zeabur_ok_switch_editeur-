✅ SOLUTION RECOMMANDÉE : NODES GOOGLE DRIVE NATIFS
═══════════════════════════════════════════════════════════════════════════════

Au lieu d'importer un JSON, vous allez créer le workflow MANUELLEMENT dans n8n
en utilisant les nodes Google Drive natifs. C'est plus simple et plus fiable !

📋 ÉTAPE 1 : CONFIGURER GOOGLE OAUTH2
═══════════════════════════════════════════════════════════════════════════════

1. Dans n8n, aller dans : Settings > Credentials

2. Cliquer sur "New Credential"

3. Chercher et sélectionner : "Google OAuth2 API"

4. Remplir les informations :
   
   Name: Google Drive CAC
   
   Client ID: 
   [VOTRE_GOOGLE_CLIENT_ID]
   
   Client Secret: 
   [VOTRE_GOOGLE_CLIENT_SECRET]
   
   Scopes (ajouter ces 2 lignes) :
   - https://www.googleapis.com/auth/drive
   - https://www.googleapis.com/auth/drive.file

5. Cliquer sur "Connect my account"

6. Autoriser l'accès dans la popup Google

7. Sauvegarder le credential

═══════════════════════════════════════════════════════════════════════════════

📋 ÉTAPE 2 : CRÉER LE WORKFLOW
═══════════════════════════════════════════════════════════════════════════════

1. Dans n8n, cliquer sur "+" pour créer un nouveau workflow

2. Nommer le workflow : "Upload Cross References CAC"

═══════════════════════════════════════════════════════════════════════════════

📋 ÉTAPE 3 : AJOUTER LE FORM TRIGGER
═══════════════════════════════════════════════════════════════════════════════

1. Ajouter un node "Form Trigger"

2. Configurer les champs du formulaire :

   Champ 1 :
   - Field Label: Client
   - Field Type: Text
   - Required: Yes
   
   Champ 2 :
   - Field Label: Exercice
   - Field Type: Text
   - Required: Yes
   
   Champ 3 :
   - Field Label: Cycle comptable
   - Field Type: Dropdown
   - Required: Yes
   - Options:
     * Capitaux propres
     * Charge d'exploitation
     * Client
     * Fournisseur
     * Immobilisations
     * Impôt et taxes
     * Personnel
     * Stock
     * Trésorerie
     * Ventes
   
   Champ 4 :
   - Field Label: Cross référence
   - Field Type: Text
   - Required: Yes
   
   Champ 5 :
   - Field Label: Document
   - Field Type: Text
   - Required: Yes
   
   Champ 6 :
   - Field Label: Fichier
   - Field Type: File
   - Required: Yes

3. Sauvegarder le node

═══════════════════════════════════════════════════════════════════════════════

📋 ÉTAPE 4 : RECHERCHER "DOSSIER CAC"
═══════════════════════════════════════════════════════════════════════════════

1. Ajouter un node "Google Drive"

2. Configurer :
   - Credential: Sélectionner "Google Drive CAC"
   - Resource: File
   - Operation: Search
   - Query: name = 'Dossier CAC' and mimeType = 'application/vnd.google-apps.folder'
   - Return All: No
   - Limit: 1

3. Connecter au Form Trigger

4. Sauvegarder

═══════════════════════════════════════════════════════════════════════════════

📋 ÉTAPE 5 : VÉRIFIER SI "DOSSIER CAC" EXISTE
═══════════════════════════════════════════════════════════════════════════════

1. Ajouter un node "IF"

2. Configurer :
   - Condition: {{ $json.data.length > 0 }}

3. Connecter au node Google Drive précédent

4. Sauvegarder

═══════════════════════════════════════════════════════════════════════════════

📋 ÉTAPE 6 : CRÉER "DOSSIER CAC" (SI N'EXISTE PAS)
═══════════════════════════════════════════════════════════════════════════════

1. Ajouter un node "Google Drive" sur la branche FALSE du IF

2. Configurer :
   - Credential: Google Drive CAC
   - Resource: Folder
   - Operation: Create
   - Name: Dossier CAC
   - Parent Folder: (laisser vide pour root)

3. Connecter à la sortie FALSE du IF

4. Sauvegarder

═══════════════════════════════════════════════════════════════════════════════

📋 ÉTAPE 7 : MERGER LES BRANCHES
═══════════════════════════════════════════════════════════════════════════════

1. Ajouter un node "Merge"

2. Configurer :
   - Mode: Append
   - Input 1: Connecter à la sortie TRUE du IF
   - Input 2: Connecter au node "Créer Dossier CAC"

3. Sauvegarder

═══════════════════════════════════════════════════════════════════════════════

📋 ÉTAPE 8 : RECHERCHER DOSSIER CLIENT
═══════════════════════════════════════════════════════════════════════════════

1. Ajouter un node "Google Drive"

2. Configurer :
   - Credential: Google Drive CAC
   - Resource: File
   - Operation: Search
   - Query: 
     name = '{{ $('Form Trigger').item.json.Client }}-{{ $('Form Trigger').item.json.Exercice }}' 
     and mimeType = 'application/vnd.google-apps.folder' 
     and '{{ $json.id }}' in parents
   - Return All: No
   - Limit: 1

3. Connecter au Merge

4. Sauvegarder

═══════════════════════════════════════════════════════════════════════════════

📋 ÉTAPE 9 : VÉRIFIER SI DOSSIER CLIENT EXISTE
═══════════════════════════════════════════════════════════════════════════════

1. Ajouter un node "IF"

2. Configurer :
   - Condition: {{ $json.data.length > 0 }}

3. Connecter au node précédent

4. Sauvegarder

═══════════════════════════════════════════════════════════════════════════════

📋 ÉTAPE 10 : CRÉER DOSSIER CLIENT + 10 CYCLES
═══════════════════════════════════════════════════════════════════════════════

1. Ajouter un node "Google Drive" sur la branche FALSE

2. Configurer pour créer le dossier client :
   - Credential: Google Drive CAC
   - Resource: Folder
   - Operation: Create
   - Name: {{ $('Form Trigger').item.json.Client }}-{{ $('Form Trigger').item.json.Exercice }}
   - Parent Folder: {{ $('Merge').item.json.id }}

3. Ajouter 10 nodes "Google Drive" en série pour créer les cycles :

   Pour chaque cycle :
   - Resource: Folder
   - Operation: Create
   - Name: [Nom du cycle]
   - Parent Folder: {{ $('Créer Dossier Client').item.json.id }}
   
   Les 10 cycles :
   1. Capitaux propres
   2. Charge d'exploitation
   3. Client
   4. Fournisseur
   5. Immobilisations
   6. Impôt et taxes
   7. Personnel
   8. Stock
   9. Trésorerie
   10. Ventes

4. Sauvegarder tous les nodes

═══════════════════════════════════════════════════════════════════════════════

📋 ÉTAPE 11 : MERGER LES BRANCHES (CLIENT)
═══════════════════════════════════════════════════════════════════════════════

1. Ajouter un node "Merge"

2. Configurer :
   - Mode: Append
   - Input 1: Connecter à la sortie TRUE du IF (client existe)
   - Input 2: Connecter au dernier node de création de cycle

3. Sauvegarder

═══════════════════════════════════════════════════════════════════════════════

📋 ÉTAPE 12 : RECHERCHER LE CYCLE SÉLECTIONNÉ
═══════════════════════════════════════════════════════════════════════════════

1. Ajouter un node "Google Drive"

2. Configurer :
   - Credential: Google Drive CAC
   - Resource: File
   - Operation: Search
   - Query: 
     name = '{{ $('Form Trigger').item.json['Cycle comptable'] }}' 
     and mimeType = 'application/vnd.google-apps.folder' 
     and '{{ $json.id }}' in parents
   - Return All: No
   - Limit: 1

3. Connecter au Merge précédent

4. Sauvegarder

═══════════════════════════════════════════════════════════════════════════════

📋 ÉTAPE 13 : UPLOADER LE FICHIER
═══════════════════════════════════════════════════════════════════════════════

1. Ajouter un node "Google Drive"

2. Configurer :
   - Credential: Google Drive CAC
   - Resource: File
   - Operation: Upload
   - File Name: [{{ $('Form Trigger').item.json['Cross référence'] }}]-{{ $('Form Trigger').item.json.Document }}
   - Binary Data: Yes
   - Binary Property: data
   - Parent Folder: {{ $json.data[0].id }}

3. Connecter au node précédent

4. Sauvegarder

═══════════════════════════════════════════════════════════════════════════════

═══════════════════════════════════════════════════════════════════════════════

2. Copier l'URL du formulaire (elle apparaît dans le Form Trigger)

3. Tester en ouvrant l'URL dans un navigateur

🔍 DÉPANNAGE
═══════════════════════════════════════════════════════════════════════════════

PROBLÈME : "Credential not found"
SOLUTION : Recréer le credential OAuth2 et le réassigner aux nodes

PROBLÈME : "Folder not found"
SOLUTION : Vérifier que le parent folder ID est correct

PROBLÈME : "File upload failed"
SOLUTION : Vérifier que le Binary Property est "data"

PROBLÈME : "Query syntax error"
SOLUTION : Vérifier les guillemets simples dans la query
