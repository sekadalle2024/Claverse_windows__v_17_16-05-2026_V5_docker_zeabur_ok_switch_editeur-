🎨 NOUVELLES FONCTIONNALITÉS DU MENU
═══════════════════════════════════════════════════════════════════════════════

Section: "Papier de travail" (📁)

1. 📐 Ajouter Schéma de calcul (Ctrl+Shift+K)
   - Déclenche la création automatique du schéma de calcul
   - Détecte la "Nature de test" dans la table 2
   - Génère le schéma approprié selon le modèle
   - Positionne le schéma au-dessus de la table principale

2. 🔄 Actualiser Schéma de calcul
   - Supprime le schéma existant
   - Retraite la table pour générer un nouveau schéma
   - Utile après modification de la "Nature de test"

3. 🗑️ Supprimer Schéma de calcul
   - Supprime le schéma du DOM
   - Supprime les données du localStorage
   - Demande confirmation avant suppression

📐 MODÈLES DE SCHÉMA SUPPORTÉS
═══════════════════════════════════════════════════════════════════════════════

Le système détecte automatiquement la "Nature de test" et applique le modèle:

✓ Validation
  Colonnes: (A) | (B) | (C) = (A) + (B) | (D) | (E) = (C) - (D)

✓ Mouvement
  Colonnes: (A) | (B) | (C) | (D) = (A) + (B) - (C) | (E) | (F) = (D) - (E)

✓ Rapprochement
  Colonnes: (A) | (B) | (C) = (A) - (B)

✓ Séparation
  Colonnes: (A) | (B) | (C) = (A) - (B)

✓ Estimation
  Colonnes: (A) | (B) | (C) = (A) * (B) | (D) | (E) = (C) - (D)

✓ Revue analytique
  Colonnes: (A) | (B) | (C) = (A) - (B)

✓ Cadrage TVA
  Colonnes: (A) | (B) = (A) * 18% | (C) | (D) | (E) | (F) = (B) - (C) - (D) - (E)

✓ Cotisations sociales
  Colonnes: (A) | (B) | (C) | (D)

✓ Vierge
  Aucune colonne (schéma vide)

✓ Modélisation (auto-détection)
  Détecte automatiquement les variables (X), (Y), (Z) dans la formule

💾 SAUVEGARDE PERSISTANTE
═══════════════════════════════════════════════════════════════════════════════

Le système utilise localStorage pour sauvegarder:
- Les valeurs des cellules du schéma de calcul
- L'association entre schéma et table principale
- Restauration automatique après actualisation de la page

Clé de stockage: "claraverse_schema_calcul_data"

🚀 UTILISATION
═══════════════════════════════════════════════════════════════════════════

1. MÉTHODE AUTOMATIQUE (recommandée)
   - Le système détecte automatiquement les tables avec "Nature de test"
   - Génère le schéma au chargement de la page
   - Aucune action manuelle nécessaire

2. MÉTHODE MANUELLE (via menu contextuel)
   a) Cliquer sur une table principale (Modelised_table)
   b) Ouvrir le menu contextuel (clic droit ou survol)
   c) Section "Papier de travail" > "Ajouter Schéma de calcul"
   d) Ou utiliser le raccourci: Ctrl+Shift+K

3. ACTUALISATION
   - Si la "Nature de test" change
   - Menu > "Actualiser Schéma de calcul"
   - Le schéma sera régénéré avec le nouveau modèle

4. SUPPRESSION
   - Menu > "Supprimer Schéma de calcul"
   - Confirmer la suppression
   - Le schéma et ses données seront effacés

🔍 DÉTECTION DE LA "NATURE DE TEST"
═══════════════════════════════════════════════════════════════════════════

Le système recherche dans la table 2:
- Une cellule contenant "Nature de test" (ou variations)
- La valeur dans la cellule suivante
- Applique le modèle correspondant

Exemples de valeurs détectées:
✓ "Rapprochement" → Modèle Rapprochement
✓ "Validation" → Modèle Validation
✓ "Mouvement" → Modèle Mouvement
✓ "Estimation" → Modèle Estimation
✓ "(X) = (Y) - (Z)" → Modèle Modélisation (auto-détecté)
