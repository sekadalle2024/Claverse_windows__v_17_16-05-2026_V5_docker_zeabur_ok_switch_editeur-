   - Conserve les données dans localStorage

═══════════════════════════════════════════════════════════════════════════════
📐 STRUCTURE
═══════════════════════════════════════════════════════════════════════════════

La Cross Référence Horizontale utilise la MÊME STRUCTURE que le Schéma de calcul:
- Même nombre de colonnes
- Pas d'en-tête de colonne
- Une seule ligne

DIFFÉRENCE VISUELLE:
- Schéma de calcul: Fond JAUNE (#fffbf0)
- Cross Référence: Fond BLEU (#f0f9ff)

═══════════════════════════════════════════════════════════════════════════════
🔢 MODÈLES SELON NATURE DE TEST
═══════════════════════════════════════════════════════════════════════════════

✓ Validation          → 5 colonnes
✓ Mouvement           → 6 colonnes
✓ Rapprochement       → 3 colonnes
✓ Séparation          → 3 colonnes
✓ Estimation          → 5 colonnes
✓ Revue analytique    → 3 colonnes
✓ Cadrage TVA         → 6 colonnes
✓ Cotisations sociales → 4 colonnes
✓ Vierge              → 0 colonnes
✓ Modélisation        → Variable (détection auto)

═══════════════════════════════════════════════════════════════════════════════
📎 FORMAT DES RÉFÉRENCES
═══════════════════════════════════════════════════════════════════════════════

FORMAT: [Lettre + Numéro]

EXEMPLES:
  [I1]      - Lead (indexation 1)
  [C100]    - Balance (indexation 10, 100)
  [F150]    - Document de test (multiples de 5 ou 10)
  [C100.1]  - Déclinaison pour mouvements
  [C100.2]  - Déclinaison pour mouvements

LETTRE ALPHABÉTIQUE:
  - Correspond au cycle comptable audité
  - Toutes les références d'un papier ont la MÊME lettre

NUMÉRO:
  - 1: Lead
  - 10, 100: Balance
  - 30, 50, 130, 150: Autres documents (multiples de 5 ou 10)

💾 SAUVEGARDE ET PERSISTANCE

CLÉ LOCALSTORAGE: claraverse_cross_ref_data

FORMAT JSON:
{
  "crossref_table_abc123_1714000000000": {
    "timestamp": 1714000000000,
    "forTable": "table_abc123",
    "cells": [
      { "row": 0, "col": 0, "value": "[C100]" },
      { "row": 0, "col": 1, "value": "[C10]" },
      { "row": 0, "col": 2, "value": "[C1]" }
    ]
  }
}

SAUVEGARDE AUTOMATIQUE:
  - Debounce de 500ms
  - Déclenchée à chaque modification de cellule
  - Restauration automatique au chargement

🔧 UTILISATION
═══════════════════════════════════════════════════════════════════════════════

1. CLIC DROIT sur une table principale (avec Conclusion/Assertion/CTR/Ecart)

2. MENU "Papier de travail" → "📎 Ajouter Cross référence horizontale"
   OU
   RACCOURCI: Ctrl+Shift+H

3. La table apparaît AU-DESSUS de la table principale

4. REMPLIR les cellules avec les références documentaires
   Exemple: [C100], [C10], [C1]

5. Les modifications sont SAUVEGARDÉES AUTOMATIQUEMENT

📊 EXEMPLE CONCRET
═══════════════════════════════════════════════════════════════════════════════

TEST: Caisse - Rapprochement

NATURE DE TEST: Rapprochement

SCHÉMA DE CALCUL:
┌─────┬─────┬──────────────────┐
│ (A) │ (B) │ (C) = (A) - (B)  │
└─────┴─────┴──────────────────┘

CROSS RÉFÉRENCE HORIZONTALE:
┌────────┬────────┬────────┐
│ [C100] │ [C10]  │ [C1]   │
└────────┴────────┴────────┘

SIGNIFICATION:
  [C100] = Balance Caisse (Cycle C, document 100)
  [C10]  = Balance auxiliaire (Cycle C, document 10)
  [C1]   = Lead Caisse (Cycle C, document 1)
