📋 PROBLÈME IDENTIFIÉ
─────────────────────────────────────────────────────────────────────────────
Le message reçu en front-end n'est pas formaté en tableaux Markdown mais en 
texte brut avec des clés-valeurs. Le format attendu par claraApiService.ts 
n'est pas respecté.

Format actuel (INCORRECT) :
  **Tableau entete**:**Etape de mission**: Programme de travail...

Format attendu (CORRECT) :
  | Rubrique | Description |
  |----------|-------------|
  | **Etape de mission** | Programme de travail |


🎯 SOLUTION
─────────────────────────────────────────────────────────────────────────────

La solution comporte 2 parties :

1. CORRIGER LE NODE CODE (après le LLM)
   → Fichier: n8n_node_code_programme_controle_comptes_FINAL.js
   
2. CORRIGER LE PROMPT LLM (optionnel mais recommandé)
   → Fichier: n8n_prompt_programme_controle_comptes_CORRIGE.txt


📦 STRUCTURE ATTENDUE PAR LE FRONT-END
─────────────────────────────────────────────────────────────────────────────

claraApiService.ts attend le FORMAT 4 :

[
  {
    data: {
      "Tableau entete": {
        "Etape de mission": "...",
        "Normes": "...",
        "Méthode": "...",
        "Reference": "..."
      },
      "Tableau processus": [
        {
          "no": 1,
          "Cycle": "...",
          "Domaines": "...",
          "Test": "...",
          "Controle audit": "...",
          "Objectif": "...",
          "Evaluation globale des risques": "...",
          "Echantillon": "..."
        }
      ]
    }
  }
]

Le service convertit ensuite cette structure en Markdown avec :
- convertHeaderTableToMarkdown() pour "Tableau entete"
- convertArrayTableToMarkdown() pour "Tableau processus"

📊 TRAITEMENT DANS claraApiService.ts
─────────────────────────────────────────────────────────────────────────────

Le service détecte automatiquement le FORMAT 4 et applique :

1. EXTRACTION (si nécessaire) :
   response.body[0].data → [{ data: {...} }]

2. DÉTECTION DU FORMAT :
   normalizeN8nResponse() identifie le FORMAT 4

3. CONVERSION EN MARKDOWN :
   convertStructuredDataToMarkdown() traite les 2 tables :
   
   a) "Tableau entete" → convertHeaderTableToMarkdown()
      Génère un tableau Markdown 2 colonnes :
      | Rubrique | Description |
      |----------|-------------|
      
   b) "Tableau processus" → convertArrayTableToMarkdown()
      Génère un tableau Markdown avec toutes les colonnes :
      | no | Cycle | Domaines | Test | ... |
      |----|-------|----------|------|-----|

4. AFFICHAGE :
   Le Markdown est rendu dans le chat Clara
