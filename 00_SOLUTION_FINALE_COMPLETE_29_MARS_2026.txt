🎯 RÉSUMÉ DU PROBLÈME
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Le frontend envoie 7 variables mais l'agent RAG n'en reçoit qu'une seule (anomalie).

Cause : 
1. Le node "Extract Variables for RAG" n'extrait pas toutes les variables
2. Le node "Alea16" écrase les variables extraites
3. Le prompt de l'agent RAG utilise une syntaxe incorrecte

✅ SOLUTION EN 3 ÉTAPES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

ÉTAPE 1 : Mettre à jour le node "Extract Variables for RAG"
─────────────────────────────────────────────────────────────────────────

1. Ouvre le node "Extract Variables for RAG" dans N8N
2. Copie TOUT le code du fichier : n8n_node_extract_variables_rag_complet.js
3. Colle-le dans le node (remplace tout le code existant)
4. Sauvegarde

Ce code extrait les 7 variables :
- command (depuis [Command])
- cycle (depuis [Cycle])
- test (depuis [test])
- reference (depuis [reference])
- nature_test (depuis [Nature de test])
- assertion (depuis [Assertion])
- anomalie (depuis [Anomalie])

ÉTAPE 2 : Configurer le node "Alea16" avec 8 champs
─────────────────────────────────────────────────────────────────────────

OPTION A - Configuration manuelle (RECOMMANDÉE) :

1. Ouvre le node "Alea16"
2. Assure-toi qu'il est de type "Edit Fields (Set)"
3. Configure 8 champs :

   Field 1: alea
   - Name: alea
   - Type: Number
   - Value: ={{ Math.floor(Math.random() * 4) + 1 }}

   Field 2: command
   - Name: command
   - Type: String
   - Value: ={{ $json.command }}

   Field 3: cycle
   - Name: cycle
   - Type: String
   - Value: ={{ $json.cycle }}

   Field 4: test
   - Name: test
   - Type: String
   - Value: ={{ $json.test }}

   Field 5: reference
   - Name: reference
   - Type: String
   - Value: ={{ $json.reference }}

   Field 6: nature_test
   - Name: nature_test
   - Type: String
   - Value: ={{ $json.nature_test }}

   Field 7: assertion
   - Name: assertion
   - Type: String
   - Value: ={{ $json.assertion }}

   Field 8: anomalie
   - Name: anomalie
   - Type: String
   - Value: ={{ $json.anomalie }}

4. Sauvegarde

OPTION B - Si tu peux copier/coller la configuration :

Le fichier n8n_node_alea16_config_complete.json contient la configuration
complète, mais N8N ne permet généralement pas d'importer directement
la configuration d'un node. Utilise l'OPTION A.

ÉTAPE 3 : Mettre à jour le prompt de l'agent RAG
─────────────────────────────────────────────────────────────────────────

1. Ouvre le node "Agent RAG"
2. Copie TOUT le contenu du fichier : n8n_prompt_agent_rag_7_variables.txt
3. Colle-le dans le champ "System Message" ou "Prompt"
4. Sauvegarde

Le prompt utilise maintenant la syntaxe N8N correcte :
- {{ $json.command }}
- {{ $json.cycle }}
- {{ $json.test }}
- {{ $json.reference }}
- {{ $json.nature_test }}
- {{ $json.assertion }}
- {{ $json.anomalie }}
