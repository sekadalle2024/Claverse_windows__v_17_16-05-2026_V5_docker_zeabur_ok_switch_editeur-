SECTION 1: DESCRIPTION DU PROBLÈME
================================================================================

PROBLÈME 1: ÉTATS DE CONTRÔLE - "ERREUR DE GÉNÉRATION"
-------------------------------------------------------
SYMPTÔME:
- Lors de l'export de la liasse, l'onglet "Contrôle de cohérence" affiche
  "Erreur de génération" pour tous les états de contrôle
- Les 16 états de contrôle ne sont pas générés correctement

COMPORTEMENT ATTENDU:
- L'onglet "Contrôle de cohérence" doit contenir 16 états de contrôle
- Chaque état doit afficher ses postes avec montants N et N-1
- Format: Sections organisées (BILAN ACTIF, BILAN PASSIF, etc.)

PROBLÈME 2: ONGLETS VIDES - AUCUNE DONNÉE REMPLIE
--------------------------------------------------
SYMPTÔME:
- Les onglets ACTIF, PASSIF, RESULTAT du fichier Excel sont vides
- Aucune valeur n'est écrite dans les cellules
- Le template Excel reste vierge

COMPORTEMENT ATTENDU:
- Les onglets ACTIF, PASSIF, RESULTAT doivent être remplis avec les valeurs
  calculées depuis les balances N et N-1
- Chaque poste doit avoir ses montants N et N-1

SECTION 2: ARCHITECTURE DU SYSTÈME
================================================================================

FLUX DE DONNÉES - EXPORT LIASSE
--------------------------------

1. FRONTEND (React/TypeScript)
   Fichier: public/ExportLiasseHandler.js
   Rôle: Déclenche l'export via bouton "Exporter Liasse Officielle"
   
   Action:
   - Récupère les résultats des états financiers depuis le state
   - Envoie une requête POST à /export-liasse/generer
   - Reçoit le fichier Excel en base64
   - Déclenche le téléchargement

2. BACKEND API (FastAPI)
   Fichier: py_backend/export_liasse.py
   Endpoint: POST /export-liasse/generer
   
   Fonction principale: remplir_liasse_officielle()
   
   Étapes:
   a) Charge le template Excel (Liasse_officielle_revise.xlsx)
   b) Détecte les onglets disponibles
   c) Convertit les données (dict→list si nécessaire)
   d) Remplit les onglets ACTIF, PASSIF, RESULTAT
   e) Génère les 16 états de contrôle
   f) Ajoute l'onglet "Contrôle de cohérence"
   g) Retourne le fichier Excel en bytes

3. MODULE ÉTATS DE CONTRÔLE
   Fichier: py_backend/generer_onglet_controle_coherence.py
   
   Fonction: generer_etats_controle_pour_export(results)
   
   Rôle:
   - Extrait les données du dictionnaire results
   - Appelle les 16 fonctions de calcul individuelles
   - Retourne une liste de 16 états de contrôle
   
   Fonction: ajouter_onglet_controle_coherence(wb, etats_controle)
   
   Rôle:
   - Crée l'onglet "Contrôle de cohérence" dans le workbook
   - Formate les 16 états avec styles Excel
   - Organise par sections (BILAN ACTIF, PASSIF, etc.)

4. MODULE CALCULS ÉTATS DE CONTRÔLE
   Fichier: py_backend/etats_controle_exhaustifs.py
   
   Contient 16 fonctions de calcul:
   - calculer_etat_controle_bilan_actif_n()
   - calculer_etat_controle_bilan_actif_n1()
   - calculer_etat_controle_bilan_actif_variation()
   - calculer_etat_controle_bilan_passif_n()
   - calculer_etat_controle_bilan_passif_n1()
   - calculer_etat_controle_bilan_passif_variation()
   - calculer_etat_controle_compte_resultat_n()

```python
def remplir_onglet_dynamique(ws, postes, col_debut, ligne_debut):
    compteur = 0
    ligne = ligne_debut
    col_n1 = chr(ord(col_debut) + 1)
    
    for poste in postes:
        montant_n = poste.get('montant_n', poste.get('montant', 0))
        if write_to_cell(ws, f'{col_debut}{ligne}', montant_n):
            compteur += 1
        
        montant_n1 = poste.get('montant_n1', 0)
        if write_to_cell(ws, f'{col_n1}{ligne}', montant_n1):
            compteur += 1
        
        ligne += 1
    
    return compteur
```

SECTION 6: HYPOTHÈSES SUR LA CAUSE DU PROBLÈME PERSISTANT
================================================================================

HYPOTHÈSE 1: PROBLÈME DE SIGNATURE DE FONCTION
-----------------------------------------------
Les fonctions calculer_etat_controle_*() attendent peut-être des paramètres
dans un format différent de ce qu'on leur passe.

À VÉRIFIER:
- Signature exacte de chaque fonction dans etats_controle_exhaustifs.py
- Format des données passées (liste vs dict, structure des postes)
- Présence de tous les champs requis (ref, libelle, montant_n, montant_n1)

HYPOTHÈSE 2: DONNÉES MANQUANTES OU MAL FORMATÉES
-------------------------------------------------
Le dictionnaire results ne contient peut-être pas toutes les données nécessaires
ou dans le bon format.

À VÉRIFIER:
- Contenu exact de results au moment de l'appel
- Présence de balance_n, balance_n1, tableau_flux_tresorerie
- Format des listes (présence de tous les champs)
- Valeurs des montants (0, None, ou valeurs réelles)

HYPOTHÈSE 3: PROBLÈME DE TEMPLATE EXCEL
----------------------------------------
Le template Excel n'a peut-être pas la structure attendue.

À VÉRIFIER:
- Noms exacts des onglets dans le template
- Numéros de lignes de départ (ligne 10 pour ACTIF, PASSIF, RESULTAT)
- Colonnes utilisées (C pour N, D pour N-1 dans ACTIF)
- Présence de cellules fusionnées qui bloquent l'écriture

HYPOTHÈSE 4: EXCEPTION SILENCIEUSE
-----------------------------------
Une exception se produit mais est capturée par le try/except et retourne
des états vides avec message d'erreur.

À VÉRIFIER:
- Logs du backend lors de l'export
- Contenu exact de l'exception (si présente)
- Stack trace complète

HYPOTHÈSE 5: PROBLÈME DE CONVERSION DICT→LIST
----------------------------------------------
La conversion bidirectionnelle dict→list→dict peut corrompre les données.

À VÉRIFIER:
- Pourquoi faire dict→list puis list→dict?
- Perte de données lors de la conversion?
- Ordre des postes préservé?

SECTION 10: POINTS D'ATTENTION CRITIQUES
================================================================================

ATTENTION 1: FORMAT DES DONNÉES
--------------------------------
⚠️ Les données peuvent être au format LISTE ou DICT selon le module qui les génère

- etats_financiers_v2.py retourne des LISTES
- Certains modules anciens retournent des DICTS
- La conversion dict→list doit être robuste
- Vérifier TOUJOURS le type avant traitement

ATTENTION 2: NOMS DES ONGLETS
------------------------------
⚠️ Les noms d'onglets varient selon la version du template

Variations possibles:
- "ACTIF" vs "BILAN-ACTIF" vs "Bilan Actif"
- "PASSIF" vs "BILAN-PASSIF" vs "Bilan Passif"
- "RESULTAT" vs "COMPTE-RESULTAT" vs "Compte de Résultat"

Solution: Recherche flexible avec .upper() et 'in'

ATTENTION 3: CELLULES FUSIONNÉES
---------------------------------
⚠️ Les cellules fusionnées nécessitent un traitement spécial

- write_to_cell() gère les cellules fusionnées
- Écrire dans la cellule en haut à gauche de la fusion
- Vérifier avec isinstance(cell, MergedCell)

ATTENTION 4: NUMÉROS DE LIGNES
-------------------------------
⚠️ Les numéros de lignes peuvent varier selon le template

Valeurs courantes:
- Ligne 10: Première ligne de données (ACTIF, PASSIF, RESULTAT)
- Ligne 1: Titre principal
- Ligne 3: Nom entreprise
- Ligne 5: Exercice

Vérifier TOUJOURS dans le template avant de coder en dur

ATTENTION 5: COLONNES N ET N-1
-------------------------------
⚠️ Les colonnes pour N et N-1 varient selon l'onglet

ACTIF:
- Colonne C: Exercice N
- Colonne D: Exercice N-1

PASSIF:
- Colonne E: Exercice N
- Colonne F: Exercice N-1

RESULTAT:
- Colonne C: Exercice N (Charges)
- Colonne D: Exercice N-1 (Charges)
- Colonne E: Exercice N (Produits)
- Colonne F: Exercice N-1 (Produits)

ATTENTION 6: GESTION DES ERREURS
---------------------------------
⚠️ Les exceptions doivent être loggées ET propagées

Mauvais:
```python
try:
    # code
except Exception as e:
    logger.error(f"Erreur: {e}")
    return []  # ❌ Masque le problème
```

Bon:
```python
try:
    # code
except Exception as e:
    logger.error(f"Erreur: {e}", exc_info=True)
    raise  # ✅ Propage l'erreur
```

ATTENTION 7: LOGS DÉTAILLÉS
----------------------------
⚠️ Les logs doivent être TRÈS détaillés pour diagnostiquer

Logs essentiels:
- Contenu de results (clés et types)
- Nombre de postes dans chaque section
- Noms d'onglets trouvés
- Nombre de cellules remplies
- Toute exception avec stack trace complète

ATTENTION 8: TESTS AVEC DONNÉES RÉELLES
----------------------------------------
⚠️ Tester avec des données RÉELLES, pas seulement des données de test

Données de test:
- Peuvent masquer des problèmes
- Ne couvrent pas tous les cas
- Peuvent avoir un format différent

Données réelles:
- Révèlent les vrais problèmes
- Testent tous les cas limites
- Format exact de production
