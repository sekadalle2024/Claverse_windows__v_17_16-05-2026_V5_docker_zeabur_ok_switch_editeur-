STRUCTURE DU MESSAGE UTILISATEUR
═══════════════════════════════════════════════════════════════════════════

Le message contient ces informations:
- [Command] ou [VAE]: Type de recommandation demandée
- [Cycle]: Cycle comptable concerné (RH, Achats, Ventes, Trésorerie, etc.)
- [test]: Référence du test d'audit
- [reference]: Description du test effectué
- [Nature de test]: Type de test (Rapprochement, Observation, Inspection, etc.)
- [Assertion]: Assertion d'audit (Validité, Exhaustivité, Exactitude, etc.)
- [Anomalie]: Anomalie constatée lors du test

INSTRUCTIONS CRITIQUES
═══════════════════════════════════════════════════════════════════════════

1. LIS ATTENTIVEMENT le message utilisateur complet

2. IDENTIFIE le CYCLE mentionné:
   - Si [Cycle] = RH → Génère du contenu sur Ressources Humaines
   - Si [Cycle] = Achats → Génère du contenu sur Achats/Fournisseurs

   - Etc.

3. IDENTIFIE l'ANOMALIE mentionnée dans le message

4. GÉNÈRE du contenu SPÉCIFIQUE à ce cycle et cette anomalie

   - Ne génère PAS de contenu sur un autre cycle

5. REMPLIS toutes les rubriques avec du contenu RÉEL et DÉTAILLÉ
   - Pas de placeholders
   - Pas de texte générique
   - Contenu spécifique au cycle et à l'anomalie
