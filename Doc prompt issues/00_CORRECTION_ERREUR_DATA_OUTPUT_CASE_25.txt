DIAGNOSTIC
═══════════════════════════════════════════════════════════════════════════

Le problème vient de la structure de données reçue par le Markdown Generator:

Structure reçue:
{
  "data": {
    "output": {
      "Etape mission - Recos contrôle interne comptable": [...]
    }
  }
}

Structure attendue par l'ancien code:
{
  "data": {
    "Etape mission - Recos contrôle interne comptable": [...]
  }
}

→ Le node "node clean1" enveloppe le JSON dans "data.output"
→ Le Markdown Generator cherche directement dans "data"
→ Il trouve un objet avec "output" au lieu d'un array

═══════════════════════════════════════════════════════════════════════════

Le code du Markdown Generator a été corrigé pour gérer 3 cas:

1. Structure avec data.output (cas actuel)
2. Structure avec data directement
3. Structure directe (sans data)
