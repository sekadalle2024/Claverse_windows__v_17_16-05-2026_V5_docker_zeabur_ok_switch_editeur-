✅ FIX APPLIQUÉ - TÉLÉCHARGEMENT N8N_DOC
=================================================================

🎯 PROBLÈME IDENTIFIÉ ET RÉSOLU !

Les logs montraient que le viewer était injecté, mais le téléchargement
ne démarrait pas. Cause : les scripts inline dans innerHTML ne s'exécutent pas.

=================================================================
🔧 SOLUTION APPLIQUÉE
=================================================================

Créé une fonction séparée `downloadAndDisplayDocument()` qui :
1. Prend le docId et l'élément viewer
2. Télécharge le document via le proxy CORS
3. Nettoie le HTML
4. L'affiche dans le conteneur

Cette fonction est appelée manuellement après l'injection du viewer.

=================================================================
💻 CODE AJOUTÉ
=================================================================

Nouvelle fonction (avant detectAndDownloadN8nDocs) :

async function downloadAndDisplayDocument(docId, viewerElement) {
    // Trouve le conteneur
    const contentDiv = viewerElement.querySelector(`#${viewerId}-content`);
    
    // Télécharge via proxy CORS
    const response = await fetch(proxyUrl + encodeURIComponent(exportUrl));
    const html = await response.text();
    
    // Nettoie et affiche
    const cleanHtml = html.replace(/<script[^>]*>.*?<\/script>/gi, '');
    contentDiv.innerHTML = cleanHtml;
}

Appel après injection :

setTimeout(() => {
    downloadAndDisplayDocument(CONFIG.docId, viewerElement);
}, 100);
