🔧 QUE FAIRE SI ÇA NE FONCTIONNE PAS ?
═══════════════════════════════════════════════════════════════════════════════

Problème 1: "0 table(s) trouvée(s)"
─────────────────────────────────────────────────────────────────────────────
Solution:
1. Vérifier dans la console: document.querySelectorAll('div.prose table')
2. Si le résultat est 0, les tables n'ont pas la classe "prose"
3. Générer de nouvelles tables dans le chat

Problème 2: Une seule table exportée
─────────────────────────────────────────────────────────────────────────────
Solution:
1. Vérifier que menu.js a été modifié
2. Rechercher "document.querySelectorAll('div.prose table')" dans exportTemplate()
3. Redémarrer le frontend: Ctrl+C puis npm run dev
4. Vider le cache du navigateur: Ctrl+Shift+R

Problème 3: Erreur dans la console
─────────────────────────────────────────────────────────────────────────────
Solution:
1. Copier l'erreur complète
2. Vérifier que SheetJS est chargé: typeof XLSX dans la console
3. Consulter: 00_CORRECTION_EXPORT_TEMPLATE_TOUTES_TABLES_09_MAI_2026.txt
