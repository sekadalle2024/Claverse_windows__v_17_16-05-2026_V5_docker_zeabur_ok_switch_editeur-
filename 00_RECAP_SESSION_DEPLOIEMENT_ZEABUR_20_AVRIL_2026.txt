└─────────────────────────────────────────────────────────────────┘

Erreur:
nginx: [emerg] "events" directive is not allowed here

Cause:
Fichier nginx.conf à la racine en conflit avec la configuration
générée dans Dockerfile.frontend

Solution:
✅ Fichier nginx.conf supprimé
✅ Configuration nginx dans Dockerfile.frontend uniquement

Statut: ✅ RÉSOLU

┌─────────────────────────────────────────────────────────────────┐
│ PROBLÈME 2: BACKEND (À CORRIGER)                                │
└─────────────────────────────────────────────────────────────────┘

Erreur:
Le service "Backend" build le frontend au lieu du backend
Logs montrent: dist/assets/*.js

Cause:
Dans Zeabur Dashboard, le service "Backend" a:
Service Name = "frontend" au lieu de "backend"

Solution:
⏳ À faire dans Zeabur Dashboard:
1. Service Backend → Settings
2. Service Name: Changer "frontend" → "backend"
3. Save → Redeploy

Temps requis: 2 minutes

Statut: ⏳ À FAIRE PAR L'UTILISATEUR

📊 ARCHITECTURE DOCKER COMPOSE
═══════════════════════════════════════════════════════════════════

docker-compose.yml définit 2 services:

┌─────────────────────────────────────────────────────────────────┐
│ Service: backend                                                │
├─────────────────────────────────────────────────────────────────┤
│ Build:                                                          │
│   context: ./py_backend                                         │
│   dockerfile: Dockerfile                                        │
│ Port: 8080                                                      │
│ Image: Python 3.11 + FastAPI + Uvicorn                         │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│ Service: frontend                                               │
├─────────────────────────────────────────────────────────────────┤
│ Build:                                                          │
│   context: .                                                    │
│   dockerfile: Dockerfile.frontend                               │
│ Port: 80                                                        │
│ Image: Node 18 (build) + Nginx (production)                    │
└─────────────────────────────────────────────────────────────────┘

Zeabur doit savoir QUEL service builder:
• Service Zeabur "Backend" → Service Name = "backend"
• Service Zeabur "Frontend" → Service Name = "frontend"

═══════════════════════════════════════════════════════════════════
🎓 POINTS CLÉS APPRIS
═══════════════════════════════════════════════════════════════════

1. Docker Compose sur Zeabur
   • Zeabur supporte Docker Compose nativement
   • Paramètre "Service Name" crucial
   • Doit correspondre au nom dans docker-compose.yml

2. Configuration Nginx
   • Éviter les fichiers nginx.conf à la racine
   • Générer la config dans le Dockerfile
   • Utiliser /etc/nginx/conf.d/default.conf

3. Multi-stage Build
   • Stage 1: Build (Node.js)
   • Stage 2: Production (Nginx)
   • Optimise la taille de l'image

4. Ports
   • Backend: 8080 (requis par Zeabur)
   • Frontend: 80 (standard Nginx)
   • Doivent correspondre aux EXPOSE dans Dockerfile

5. Logs de Diagnostic
   • Backend correct: "pip install", "uvicorn running"
   • Frontend correct: "npm ci", "vite build", "nginx"
   • Logs incorrects = mauvaise configuration
