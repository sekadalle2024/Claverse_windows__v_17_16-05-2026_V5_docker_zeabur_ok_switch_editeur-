2. NORMALISATION REGEX IMPLÉMENTÉE
═══════════════════════════════════════════════════════════════════════════════

Fonction normalizeValue() créée pour gérer TOUTES les variations:

ÉLEVÉ:
  ✓ Élevé, élevé, ÉLEVÉ, ELEVE, eleve, Elevé
  ✓ High, high, HIGH
  ✓ Fort, fort, FORT, Forte, forte

MOYEN:
  ✓ Moyen, moyen, MOYEN
  ✓ Moyenne, moyenne, MOYENNE
  ✓ Medium, medium, MEDIUM
  ✓ Modéré, modéré, MODÉRÉ, Modere, modere
  ✓ Mod, mod, MOD

FAIBLE:
  ✓ Faible, faible, FAIBLE
  ✓ Low, low, LOW
  ✓ Toute autre valeur par défaut

TRAITEMENT:
  1. Conversion en minuscules
  2. Suppression des accents (NFD normalization)
  3. Regex pattern matching
  4. Retour valeur normalisée: 'Élevé' | 'Moyen' | 'Faible'

3. INTÉGRATION AUTOMATIQUE DES RISQUES
═══════════════════════════════════════════════════════════════════════════════

FONCTION: calculerRepartitionRisques()

AVANT:
  const key = `${risque.Probabilite}-${risque.Impact}`;
  ❌ Problème: "Elevé-Moyen" ≠ "Élevé-Moyen" ≠ "eleve-moyen"

APRÈS:
  const probNormalized = normalizeValue(risque.Probabilite);
  const impactNormalized = normalizeValue(risque.Impact);
  const key = `${probNormalized}-${impactNormalized}`;
  ✅ Solution: Toutes les variations → "Élevé-Moyen"

RÉSULTAT:
  ✓ Tous les risques sont automatiquement placés dans la bonne cellule
  ✓ Peu importe la casse ou les accents dans les données source
  ✓ Aucune perte de risque due à des variations d'écriture
