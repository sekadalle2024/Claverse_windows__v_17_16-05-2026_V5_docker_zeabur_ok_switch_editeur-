🔧 SOLUTIONS APPLIQUÉES
=================================================================

FIX 1 : EMPÊCHER LES DUPLICATIONS
----------------------------------
Ajout d'un marqueur sur la table elle-même :

table.dataset.gdriveProcessed = 'true';

Avant de traiter une table, on vérifie :
- Si elle a déjà le marqueur → skip
- Si elle a déjà un viewer après elle → skip

FIX 2 : DÉSACTIVER LE MUTATIONOBSERVER
---------------------------------------
Le MutationObserver créait des boucles infinies :
- Il détectait les changements du DOM
- Relançait la détection
- Qui ajoutait des éléments
- Qui déclenchait à nouveau l'observer
- → BOUCLE INFINIE

Solution : DÉSACTIVÉ complètement
Le script s'exécute maintenant UNE SEULE FOIS au chargement.

FIX 3 : PROTECTION CONTRE LES ERREURS
--------------------------------------
Ajout d'un try-catch global pour éviter que les erreurs
ne plantent toute la page.

🎯 POURQUOI LE MUTATIONOBSERVER ÉTAIT PROBLÉMATIQUE
=================================================================

Le MutationObserver observait TOUS les changements du DOM.
Quand on injectait le viewer :
1. MutationObserver détecte l'ajout
2. Relance detectAndDownloadN8nDocs()
3. Qui injecte un nouveau viewer
4. Qui déclenche à nouveau le MutationObserver
5. → BOUCLE INFINIE

Résultat :
- Duplications infinies
- Surcharge mémoire
- Plantage de la page

Solution :
- Désactiver le MutationObserver
- Exécuter UNE SEULE FOIS au chargement
- Marquer les tables traitées
