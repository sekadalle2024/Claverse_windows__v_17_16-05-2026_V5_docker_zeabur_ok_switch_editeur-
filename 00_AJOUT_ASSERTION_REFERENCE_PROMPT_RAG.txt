MODIFICATION 1 : ÉTAPE 1 - Ajout du contexte des variables
─────────────────────────────────────────────────────────────────────────

AVANT :
Analyse :
- Quelle est la nature de l'opération sous-jacente ?
  → Analyser la [référence] et l'[anomalie] pour identifier l'opération

APRÈS :
CONTEXTE DU CONTRÔLE :
- [reference] = "{{ $json.reference || "Non fourni" }}" 
  → Décrit le test/contrôle effectué (ex: "test sur la validation du compte caisse")
- [assertion] = "{{ $json.assertion || "Non fourni" }}"
  → Assertion d'audit testée (existence, exhaustivité, validité, exactitude, etc.)
- [anomalie] = "{{ $json.anomalie || "Non fourni" }}"
  → Anomalie détectée lors du contrôle

Analyse :
- Quelle est la nature de l'opération sous-jacente ?
  → Analyser la [reference] et l'[anomalie] pour identifier l'opération
- Quels sont les flux économiques ?
  → Déduire des informations de la [reference] et du [cycle]
- Quelle est l'écriture de régularisation probable ?
  → Analyser l'[anomalie] pour déterminer la régularisation
- Quelle assertion est testée ?
  → Utiliser [assertion] pour orienter les recherches (ex: validité → vérifier pièces justificatives)

Variable [reference] :
─────────────────────────────────────────────────────────────────────────
Contenu : "test sur la validation du compte caisse"
Utilisation :
  - RECHERCHE 1 : Affiner la recherche du compte principal
  - RECHERCHE 2 : Identifier les comptes de charges/produits concernés

Variable [assertion] :
─────────────────────────────────────────────────────────────────────────
Contenu : "validité"
Utilisation :
  - RECHERCHE 4 : Orienter la régularisation selon l'assertion
  - RECHERCHE 6 : Identifier les pièces justificatives requises

Exemples d'assertions :
  - existence → Vérifier que le compte existe
  - exhaustivité → Vérifier que toutes les opérations sont enregistrées
  - validité → Vérifier les pièces justificatives
  - exactitude → Vérifier les montants et calculs
  - imputation → Vérifier l'affectation aux bons comptes
