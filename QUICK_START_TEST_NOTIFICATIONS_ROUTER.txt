🎯 OBJECTIF
─────────────────────────────────────────────────────────────────────────────
Tester les notifications qui affichent le case, la route et l'URL utilisés
par le router n8n dans claraApiService.ts

⚡ DÉMARRAGE RAPIDE (3 ÉTAPES)
─────────────────────────────────────────────────────────────────────────────

1️⃣ DÉMARRER L'APPLICATION
   ─────────────────────────
   npm run dev

2️⃣ OUVRIR L'APPLICATION
   ─────────────────────
   http://localhost:5173

3️⃣ TESTER UN MESSAGE
   ──────────────────
   Dans le chat, envoyer : "CIA Cours"
   
   ✅ Résultat attendu :
   Une notification apparaît en haut à droite :
   
   ┌─────────────────────────────────────────────┐
   │ 🔀 Router n8n - Case 11                     │
   │                                             │
   │ Route: cia_cours                            │
   │ URL: http://localhost:5678/webhook/...      │
   └─────────────────────────────────────────────┘

🔍 VÉRIFICATION CONSOLE
─────────────────────────────────────────────────────────────────────────────
Ouvrir la console développeur (F12) et chercher :

🔀 Router → Case XX : route_key
🔔 Notification envoyée: { case: "Case XX", route: "...", url: "..." }

✅ CRITÈRES DE SUCCÈS
─────────────────────────────────────────────────────────────────────────────
✓ La notification apparaît en haut à droite
✓ Le titre affiche "🔀 Router n8n - Case XX"
✓ Le message affiche la route et l'URL
✓ La notification reste visible 8 secondes
✓ Les logs console confirment l'envoi

❌ PROBLÈMES POSSIBLES
─────────────────────────────────────────────────────────────────────────────

Problème : Notification ne s'affiche pas
Solution : Vérifier que notificationService est bien importé

Problème : URL incorrecte affichée
Solution : Vérifier la logique de détection dans getN8nEndpointWithInfo()

Problème : Case incorrect détecté
Solution : Vérifier l'ordre des conditions if/else dans getN8nEndpointWithInfo()

🎨 PERSONNALISATION
─────────────────────────────────────────────────────────────────────────────
Pour modifier la durée d'affichage (actuellement 8000ms = 8s) :

Dans claraApiService.ts, ligne ~1250 :
addInfoNotification(
  `🔀 Router n8n - ${routingInfo.caseName}`,
  `Route: ${routingInfo.routeKey}\nURL: ${resolvedEndpoint}`,
  8000  // ← Modifier cette valeur (en millisecondes)
);
