PROBLÈME RÉSOLU
───────────────────────────────────────────────────────────────────────────
Les tables 2-6 s'affichaient mais les en-têtes n'étaient PAS en gras:
- "Intitule", "Observation", "Constat", etc. en texte normal
- Pas de distinction visuelle

SOLUTION APPLIQUÉE
───────────────────────────────────────────────────────────────────────────
Utiliser un TABLEAU MARKDOWN 1 COLONNE avec des TIRETS SIMPLES pour le séparateur.

Cette approche garantit:
✅ En-têtes en gras (Markdown `**...**`)
✅ Séparateur correct (tirets simples `-` au lieu de `─`)
✅ Cohérence visuelle avec la table 1
✅ Compatible avec ReactMarkdown
✅ Pas besoin de modifier le workflow n8n

CORRECTION CRITIQUE
───────────────────────────────────────────────────────────────────────────
Le premier essai utilisait des caractères Unicode `─` pour le séparateur,
ce qui cassait la structure du tableau. Corrigé en utilisant des tirets
simples `-` comme requis par la spécification Markdown.

FICHIER MODIFIÉ
───────────────────────────────────────────────────────────────────────────
src/services/claraApiService.ts
→ Fonction convertHeaderTableToMarkdown() (ligne ~650)

CHANGEMENT EFFECTUÉ

AVANT (HTML avec styles inline):
```typescript
let md = `\n<h3 style="font-weight: bold;">Intitule</h3>\n\n`;

```

APRÈS (Tableau Markdown 1 colonne):
```typescript
let md = `\n| **${headerText}** |\n`;
md += `|${'-'.repeat(headerText.length + 6)}|\n`;  // ✅ Tirets simples
md += `| ${cellValue.replace(/\n\n/g, '<br><br>')} |\n\n`;
```

IMPORTANT: Utiliser des TIRETS SIMPLES `-` pour le séparateur, pas des
caractères Unicode `─` qui cassent la structure du tableau Markdown.
