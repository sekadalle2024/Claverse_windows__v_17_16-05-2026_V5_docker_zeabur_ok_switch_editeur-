📊 STRUCTURE DES TEMPLATES
═══════════════════════════════════════════════════════════════════════════════

Tous les templates suivent la structure JSON existante :

{
  "Etape mission - [Nom]": [
    {
      "table 1": { ... }  // En-tête ou données
    },
    {
      "table 2": [ ... ]  // Tableau de données
    }
  ]
}

Cette structure est compatible avec la méthode convertStructuredDataToMarkdown()
qui gère déjà ce format.

📝 NOTES IMPORTANTES

1. HIÉRARCHIE RESPECTÉE
   Les cases 35-43 sont placés AVANT les cases 2-10 dans le code, mais
   APRÈS les cases 11-34 pour respecter la hiérarchie de priorisation.

2. PAS D'APPEL N8N
   Ces cases ne font aucun appel HTTP vers n8n. Tout est généré localement
   pour des performances optimales.

   Les templates utilisent le même format JSON que les autres cases,
   garantissant une cohérence dans l'affichage.

4. EXTENSIBILITÉ
   Pour ajouter un nouveau template, il suffit de :
   - Ajouter une condition dans getN8nEndpointWithInfo()
   - Ajouter le routeKey dans resolveEndpointFromRouteKey()
   - Ajouter le template JSON dans generateTemplateTable()
