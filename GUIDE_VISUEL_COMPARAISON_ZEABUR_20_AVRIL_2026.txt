💡 ASTUCE MNÉMOTECHNIQUE
═══════════════════════════════════════════════════════════════════

Pour ne plus oublier:

Service Zeabur "Backend"  → Service Name = "backend"
Service Zeabur "Frontend" → Service Name = "frontend"

C'est comme une étiquette sur une boîte:
📦 Boîte "Backend"  → Étiquette "backend"
📦 Boîte "Frontend" → Étiquette "frontend"

Si vous mettez l'étiquette "frontend" sur la boîte "Backend",
vous allez ouvrir la mauvaise boîte ! 📦❌

═══════════════════════════════════════════════════════════════════
🆘 SI VOUS ÊTES BLOQUÉ
═══════════════════════════════════════════════════════════════════

1. Vérifier que vous êtes sur le bon service:
   → URL doit être pybackend.zeabur.app (pas prclaravi)

2. Vérifier que Build Method = "Docker Compose":
   → Pas "Dockerfile" ou "Buildpack"

3. Vérifier que le repo et la branche sont corrects:
   → Repository: github.com/votre-username/claraverse
   → Branch: master

4. Si le champ "Service Name" n'existe pas:
   → Vérifier que Build Method = "Docker Compose"
   → Le champ apparaît seulement avec Docker Compose

5. Si le problème persiste après correction:
   → Supprimer le service et le recréer
   → Ou utiliser votre ancien repo backend séparé
