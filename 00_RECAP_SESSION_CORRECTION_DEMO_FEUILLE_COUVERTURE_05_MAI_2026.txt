📋 CONTEXTE
-----------
L'étape "Feuille couverture" de E-audit Pro dispose d'un mode Demo, mais la
ligne [Demo] = Activate est ajoutée en PREMIÈRE position au lieu de la DERNIÈRE.

🎯 OBJECTIF
-----------
Déplacer [Demo] = Activate de la première ligne vers la dernière ligne
(après [Nb de lignes] = X)

🔍 ANALYSE
----------
Le problème vient de la définition du mode Demo dans DemarrerMenu.tsx:

  { id: 'demo', label: 'Demo', prefix: '[Demo] = Activate\n' }

Le "prefix" ajoute la ligne AU DÉBUT de la commande.
Il faut utiliser un "suffix" pour l'ajouter À LA FIN.

✅ SOLUTION APPLIQUÉE
---------------------

1. Ajout de la propriété "suffix" à l'interface ModeItem
2. Modification du mode Demo pour utiliser suffix au lieu de prefix
3. Modification de la logique de génération de commande

📊 MODIFICATIONS TECHNIQUES
---------------------------

Interface ModeItem:
  AVANT: prefix?: string;
  APRÈS: prefix?: string; suffix?: string;

Mode Demo:
  AVANT: { id: 'demo', label: 'Demo', prefix: '[Demo] = Activate\n' }
  APRÈS: { id: 'demo', label: 'Demo', prefix: '', suffix: '\n[Demo] = Activate' }

Logique de génération:
  AVANT: mode.prefix + activeEtape.command
  APRÈS: mode.prefix + activeEtape.command + (mode.suffix || '')
