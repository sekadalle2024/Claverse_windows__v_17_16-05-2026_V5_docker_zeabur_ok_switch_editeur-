🔍 RÉPONSE À VOS QUESTIONS
═══════════════════════════════════════════════════════════════════

**Q1: Avoir un repo GitHub qui ne contient pas de backend ?**

R: Vous avez 2 choix:

A) Créer un repo séparé "claraverse-frontend" (Option 1)
   → Plus simple, comme vous avez fait pour le backend
   → Recommandé si vous voulez la simplicité

B) Utiliser le repo complet (Option 2)
   → Spécifier Dockerfile.frontend
   → Root Directory = /
   → Zeabur va builder uniquement le frontend

**Q2: Spécifier le Root Directory ?**

R: OUI, mais laisser "/" (racine)

Le Root Directory indique où se trouve le code à builder.
Dans votre cas:
- Dockerfile.frontend est à la racine
- Le code frontend (src/) est à la racine
- Donc Root Directory = /

**Q3: Docker Compose ET Dockerfile seront exécutés ?**

R: NON ! Zeabur exécute SOIT l'un SOIT l'autre:

┌─────────────────────────────────────────────────────────────────┐
│ Build Method = "Dockerfile"                                     │
│ → Zeabur utilise UNIQUEMENT le Dockerfile spécifié             │
│ → docker-compose.yml est IGNORÉ                                │
│                                                                 │
│ Build Method = "Docker Compose"                                │
│ → Zeabur utilise UNIQUEMENT docker-compose.yml                 │
│ → Dockerfile individuel est IGNORÉ                             │
└─────────────────────────────────────────────────────────────────┘

**Recommandation:**
Utilisez Build Method = "Dockerfile" pour le frontend
(comme vous avez fait pour le backend)

⚠️ ERREURS À ÉVITER
═══════════════════════════════════════════════════════════════════

❌ NE PAS utiliser Build Method = "Docker Compose"
   → Vous avez déjà le backend séparé
   → Ça va créer de la confusion

❌ NE PAS oublier de spécifier "Dockerfile.frontend"
   → Sinon Zeabur va chercher "Dockerfile" par défaut
   → Ça va essayer de builder le backend

❌ NE PAS mettre Root Directory = "src"
   → Le Dockerfile.frontend est à la racine
   → Il a besoin d'accéder à src/, public/, etc.

✅ FAIRE:
   → Build Method = "Dockerfile"
   → Dockerfile = "Dockerfile.frontend"
   → Root Directory = "/"
