  DIAGNOSTIC - PROBLÈME PERSISTANT FORMATAGE TABLEAUX
  Date: 31 Mars 2026
═══════════════════════════════════════════════════════════════════════════════

🔴 SYMPTÔME
─────────────────────────────────────────────────────────────────────────────
Le message reçu en front-end affiche du texte brut au lieu de tableaux Markdown:

  **Tableau entete**:**Etape de mission**: Programme de travail...

Au lieu de:

  | Rubrique | Description |
  |----------|-------------|
  | **Etape de mission** | Programme de travail |


🔍 ANALYSE DU PROBLÈME
─────────────────────────────────────────────────────────────────────────────

Le problème peut avoir 3 causes possibles :

1. ❌ Le node Code ne retourne PAS le FORMAT 4 attendu
   → Structure attendue : [{ data: {...} }]
   
2. ❌ claraApiService.ts ne DÉTECTE PAS le FORMAT 4
   → Il utilise convertGenericStructureToMarkdown() au lieu de 
     convertStructuredDataToMarkdown()
   
3. ❌ Le webhook n8n enveloppe la réponse dans response.body[0].data
   → Le node Code ne gère pas cette enveloppe

🔧 SOLUTIONS SELON LE DIAGNOSTIC
─────────────────────────────────────────────────────────────────────────────

SOLUTION 1 : Le webhook retourne response.body[0].data
───────────────────────────────────────────────────────

Si le test webhook montre cette structure :
  {
    "response": {
      "body": [
        {
          "data": {
            "Tableau entete": {...},
            "Tableau processus": [...]
          }
        }
      ]
    }
  }

ALORS :
→ Remplacer le node Code par : n8n_node_code_programme_controle_ULTRA_ROBUSTE.js
→ Ce code gère automatiquement cette enveloppe


SOLUTION 2 : Le node Code ne retourne pas le bon format
────────────────────────────────────────────────────────

Si les logs du node Code montrent une erreur de parsing ou une structure 
incomplète :

ALORS :
→ Vérifier le prompt du LLM
→ S'assurer qu'il retourne EXACTEMENT :
  [
    {"Tableau entete": {...}},
    {"Tableau processus": [...]}
  ]
  
→ Ajouter au début du prompt LLM le contenu de :
  n8n_prompt_programme_controle_comptes_CORRIGE.txt


SOLUTION 3 : claraApiService.ts ne détecte pas le FORMAT 4
───────────────────────────────────────────────────────────

Si les logs du navigateur montrent "Format non reconnu" :

ALORS :
→ Vérifier dans claraApiService.ts la fonction normalizeN8nResponse()
→ Chercher la section "FORMAT 4"
→ Vérifier que la condition de détection est correcte :

  if (Array.isArray(result) && result.length > 0 && result[0].data) {
    // FORMAT 4 détecté
  }
