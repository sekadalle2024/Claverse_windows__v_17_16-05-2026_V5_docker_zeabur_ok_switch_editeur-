PROBLÈME 1: DÉTECTION DES TABLES
═══════════════════════════════════════════════════════════════

Symptôme:
- Extension Chrome détecte 24 tables
- Menu contextuel affiche "Aucune table trouvée" (0 table)
- Export impossible

Cause:
- Le code cherchait d'abord un conteneur avec querySelector('div.prose')
- Puis cherchait les tables dedans avec 'table'
- Le conteneur n'était pas trouvé correctement
- Résultat: 0 table détectée

Solution:
- Utilisation directe du sélecteur global 'div.prose table'
- Plus besoin de chercher un conteneur spécifique
- Détecte maintenant les 24 tables correctement

Fichier modifié:
- public/menu.js (ligne ~7350)

═══════════════════════════════════════════════════════════════
PROBLÈME 2: CONTENU INCOMPLET
═══════════════════════════════════════════════════════════════

Symptôme:
- Contenu incomplet dans le rapport Word exporté
- Seule la première ligne des cellules était extraite
- Les paragraphes suivants étaient perdus

Exemple:
- Table avec 3 paragraphes dans "Description"
- Export ne contenait que le 1er paragraphe
- Les 2 autres paragraphes étaient perdus

Solution:
1. Amélioration de extractTableDataOptimized()
   - Préserve les retours à la ligne multiples
   - Maintient la structure du texte

2. Nouvelle fonction extractFullCellContent()
   - Extrait TOUTES les lignes d'une cellule
   - Joint les lignes avec double retour à la ligne
   - Gère les tables à 1 ou 2 colonnes

3. Modification des fonctions de collecte
   - collectFrapPoints()
   - collectRecosRevisionPoints()
   - collectRecosControleInternePoints()
   - Utilisent maintenant extractFullCellContent()

Fichiers modifiés:
- public/menu.js (lignes 4238, 7419, 7499, 7573)
